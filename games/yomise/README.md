# よみせ — 縁日屋台経営

夏祭りの夜店通りを舞台にしたアイドル(放置)×タップ型の屋台経営ゲーム。

## プレイURL

https://nrtkumi.github.io/games/yomise/

## 対応環境

- スマートフォン(iOS Safari / Android Chrome) — タッチ操作
- PC ブラウザ(Chrome / Firefox / Safari) — マウスクリックで同様に操作可
- 単一HTMLファイル、インストール不要

## 遊び方・ルール

### 基本操作

| 操作 | 効果 |
|------|------|
| 屋台アイコンをタップ | その屋台でコインを1回稼ぐ(レベル依存) |
| 「出店」ボタン | 新しい屋台を開く(初回のみ) |
| 「強化」ボタン | 屋台のレベルを上げ、稼ぎをアップ |
| 「店員」ボタン | 店員を雇い、放置でも自動でコインを稼ぐ |
| 「のれん分け」ボタン | 屋台をリセットして永続の常連ボーナスを取得 |

### 屋台一覧

| 屋台 | アイコン | タップ基礎値 | 自動基礎値(/秒) | 出店コスト | 店員コスト |
|------|---------|-------------|----------------|-----------|-----------|
| たこ焼き | 🐙 | 1 | 0.2 | 15 | 80 |
| りんご飴 | 🍎 | 3 | 0.6 | 80 | 400 |
| わたあめ | 🍬 | 8 | 1.5 | 350 | 1,800 |
| かき氷 | 🧊 | 20 | 4.0 | 1,500 | 8,000 |
| 焼きそば | 🍜 | 60 | 12.0 | 7,000 | 40,000 |
| たい焼き | 🐟 | 200 | 40.0 | 35,000 | 200,000 |
| ベビーカステラ | 🧁 | 800 | 160.0 | 200,000 | 1,200,000 |
| いか焼き | 🦑 | 3,000 | 600.0 | 1,200,000 | 8,000,000 |

- タップ/自動の稼ぎはレベルに比例して増加
- コストは強化するたびに約 ×1.15 ずつ増加
- 常連ボーナス(のれん分け後)はこれら全ての値に乗算される

### 放置・帰還報酬(おかえりコイン)

ゲームを閉じている間も、店員を雇った屋台は自動でコインを稼ぎ続ける。
再度開くと「おかえり! 留守中に X コイン稼ぎました」のポップアップが表示される。

- 上限: 2時間分のコイン(それ以上は蓄積しない)
- 効率: オフライン中は通常の 50% レートで稼ぐ

### のれん分け(プレスティージュ)

累計獲得コインが閾値を超えると「のれん分け」が可能になる。

| 回数 | 累計コイン閾値 | 常連ボーナス倍率 |
|------|--------------|---------------|
| 1回目 | 5,000 | ×1.25 |
| 2回目 | 17,500 | ×1.50 |
| 3回目 | 61,250 | ×1.75 |
| 以降 | ×3.5 ずつ増加 | +0.25 ずつ増加 |

のれん分け後は屋台がリセットされるが、常連ボーナスは永続する。

## 作成の背景

2026年のブラウザ/モバイルゲームにおける放置(アイドル)ジャンルは、核になるループ構造の明快さと「帰ってきたときの報酬感」で差がつく。

**Action → Reward → Investment の三角形:** GameAnalytics によれば、成功したアイドルゲームはこの三角形を明確に設計している。タップ(Action)でコインを得て(Reward)、屋台強化に投資する(Investment)という本作のループはそのまま実装した。([How to make an idle game: adjust for fun, not theory](https://www.gameanalytics.com/blog/how-to-make-an-idle-game-adjust))

**「帰ってきたときの報酬」習慣ループ:** 離脱→再訪時に蓄積報酬を提示する設計は、特に長い待ち時間(電車・行列)のあとにスマホを開く文脈と相性がよく、リテンション向上に有効と知られている。オフライン報酬に上限(2時間)を設けることで、毎日の帰還動機を維持しながら、何日も放置したときの進行崩壊を防いでいる。([Deconstructing Indie Success: Idle & Roguelike Games](https://medium.com/@hrldthomas/deconstructing-indie-success-a-growth-pms-guide-to-idle-and-roguelike-games-6a2f982aacaf))

**サムゾーン設計:** 全操作ボタンを下部に集中させた縦スクロール型UIは、片手親指で届く「親指ゾーン」理論に基づく。([Designing for One-Hand Usage](https://www.smashingmagazine.com/2020/02/design-mobile-apps-one-hand-usage/))

## 技術構成

- 単一 HTML ファイル(CSS・JS インライン)、外部依存なし(フォントは Google Fonts CDN)
- UI: DOM ベース(屋台カード・ボタン)+ Canvas バックドロップ(夜空・提灯・星・パーティクル)
- 音声: Web Audio API でリアルタイム合成(外部音源なし)。iOS 消音スイッチ対策として無音 WAV ループ + AudioContext 自動復帰を実装
- 永続化: localStorage(try/catch フォールバック)。visibilitychange / pagehide で都度セーブ
- タッチ: Pointer Events。ダブルタップズーム防止(iOS の user-scalable=no 無視対策)
- PC 対応: `position:fixed; left:50%; transform:translateX(-50%)` で縦長カラムに制限

**デザインコンセプト:** 夏祭りの夜店通り。深藍(indigo)の夜空、提灯の赤/橙/金、のれんの交互カラー、Reggae One(見出し)+ Zen Maru Gothic(本文)。グラスモーフィズムや紫グラデーションを意識的に避け、縁日らしい温かみのある色合いに。

- 自動収入(cps)計算

- オフライン報酬計算(5分 × 50% レート)

### 3. 実ブラウザ E2E テスト(Puppeteer)

`puppeteer-core` + ローカル Chrome + `python3 -m http.server` で HTTP サーブして実施。
モバイルビューポート(390×844、hasTouch:true、deviceScaleFactor:2)。

**テスト結果(9件全 PASS):**
- Zero pageerror / Zero JS console error
- Canvas が描画ピクセルを持つ
- タイトル画面 → はじめる でゲーム開始
- タップでコイン獲得
- 強化ボタンでレベルアップ
- localStorage にセーブ → リロード後にコイン復元
- 5分前 `lastSeen` を設定した状態でリロード → おかえりポップアップ表示
- 2秒待機で自動収入が増加
- のれん分けボタンが条件達成時に有効化

### 教訓

- **「たこ焼きは最初から出店済み」設計:** 初期コイン 0 で全屋台がロック状態だとタップして稼ぐ手段もないデッドロックになる。最初の屋台を Lv.1 スタートにして即タップ → 稼ぎ → 出店の流れを保証した
- **localStorage の競合:** E2E テストでゲームが `visibilitychange` 時に自動セーブするため、テスト中に localStorage を書き換えてもリロード前に上書きされる問題が発生。Puppeteer の `createBrowserContext()` でインコグニト隔離することで解決した
- **OGP のサムゾーンクリップ:** `deviceScaleFactor:2` + 600×315 px クリップで 1200×630 px の JPEG を生成。53 KB に収まった

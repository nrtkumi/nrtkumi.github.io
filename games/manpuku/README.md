# 満腹食堂 — 爆速まかないアクション

**伝票の食材**を、棚から大急ぎでかき集めて配膳しろ。客の腹が減りきる前に手際よくさばくほど**手際メーター**が燃え、**FEVER**で売上が倍に。実在料理の写真と食材画像で遊ぶ、昭和食堂の早さばきアクション。

- **プレイ**: https://nrtkumi.github.io/games/manpuku/
- **対応環境**: スマホ縦画面ファースト(タッチ操作)。モダンブラウザ。
- **デザインコンセプト**: 昭和の大衆食堂。のれんの朱、琺瑯看板の黄、タイル壁。極太の角丸ポスター体(Dela Gothic One)。明るく賑やかで、他2作(暗いバー/賭場)と対になる配色。

## 遊び方

1. 客が料理を注文する。**必要な食材(伝票)は常に表示**されている — 隠れない。覚える必要はなく、**速く見つける**ゲーム。
2. 棚(パントリー)から、伝票にある食材だけを大急ぎでタップ。全部そろえば自動で配膳。
3. ニセ食材(デコイ)をタップするとミス。手際が落ち、客の待ち時間が削られる。
4. 速く・コンボを切らさず配膳するほど**手際メーター**が溜まり、満タンで **FEVER(売上×2)**。
5. 客の待ち時間が尽きると帰られて信用(🍜)が減る。数人さばくごとに**まかない(強化)**を選べる。
6. 信用が尽きたら本日の営業終了。売上が記録される。

### まかない(強化)

| 名前 | 効果 |
|---|---|
| 🍲 出汁 | 客の待ち時間がいつも長くなる |
| 🍚 大盛り | 売上がすべて +30% |
| 📋 品書き | 毎回、必要な棚を1つ光らせる |
| 🍶 常連客 | 1皿につき最初のミスを見逃す |
| 🏮 招き猫 | 信用＋1、最大も＋1 |
| 🔪 仕込み | 棚のニセ食材(デコイ)が減る |
| 💨 韋駄天 | 手際メーターが2倍たまる(FEVERしやすい) |

## 作成の背景

`/deep-research` による市場調査(2026-06)を踏まえた。

- **コンボ/FEVER/速さ勝負の短いループ**が中毒性の核という結論に沿い、「見つけて・タップして・配膳」の純アクションに手際メーターとFEVERを乗せた。出典: [gamigion: Top 10 Hybridcasual Games Q1 2025](https://www.gamigion.com/top-10-hybridcasual-games-in-q1-2025-the-great-puzzle-takeover/)
- 姉妹作シズク(記憶系)とあえて**同じ「食材タップ」操作**にしつつ、**レシピを隠さず常に見せる**ことで「記憶」ではなく「反射・速さ」のゲームに振った。隠す/見せるの一点でジャンルが割れることの実証。
- 既存作のドラフト(KAITEN)や物理マージ(suika)とは別の、**時間圧+行列さばき**のアクション。

データは **[TheMealDB](https://www.themealdb.com/api.php)** の公開API。`search.php?f={letter}` で実在料理(写真・材料を一括)を取得し、食材サムネイル(`/images/ingredients/{name}-Small.png`)を棚タイルに使う。CORS対応・キー不要(テストキー `1`)・無料。

> 注意: TheMealDBのテストキー `1` は開発・教育・個人利用向け。アプリストア公開にはPremiumキーが必要(本サイトは個人のホビー用途のため可)。

## 技術構成

- 単一HTML(外部ライブラリなし)。レイアウトはDOM、待ち時間カウントダウン・手際メーター減衰・FEVERタイマー・紙吹雪粒子はすべて `requestAnimationFrame` 1本で駆動(ループはスタート前から回り、触る状態は宣言時に初期化)。
- 料理は起動時に `search.php?f=` を数文字ぶん取り寄せて約70〜100品をキャッシュ(`localStorage`)。次回は即開店&裏で補充。通信全滅時は内蔵の定番10品(テキスト)にフォールバック。
- 料理写真・食材画像とも `onerror` で絵文字フォールバック。
- iOS消音スイッチ対策(無音WAV + `resume()` + `onstatechange`)。効果音はWeb Audioでリアルタイム合成。
- スマホ対策: viewport固定、セーフエリア、Pointer Events、ダブルタップズーム無効、`max-width: min(480px,64vh)` の縦カラムでPCワイド画面対応。
- OGP一式(絶対URL)。`assets/ogp.jpg` は 1200×630。

## テスト

1. `node --check` で構文チェック。
2. puppeteer-core + ローカルChrome + モバイルビューポート(`hasTouch`)で実機相当E2E: のれん→開店→23人ぶんの配膳・コンボ・まかない選択(3回)・難易度上昇まで自動プレイ。`pageerror`/console エラー0(favicon除く)、非faviconの404が0件、食材画像404は `onerror` で吸収。
3. ローカルHTTPサーバ(`127.0.0.1`)経由でテスト(`file://` の外部画像によるcanvas汚染を回避)。

## 開発で得た教訓

- 同じ「食材タップ」でも、**伝票を常時表示にする**だけで記憶ゲー(シズク)が反射ゲーになる。1サイト内で素材APIを変え(カクテル/料理)、操作を共有しつつ体験を割るのは有効な作り分け。
- `search.php?f={letter}` は**1リクエストで数十品(材料込み)**が返るので、`random.php` を何十回も叩くより圧倒的に軽い。料理データは文字検索でまとめ取りするのが正解。

# オーラキャッチ!(AURA CATCH)

縮むリングがキャラにピッタリ重なる瞬間にタップして、ヘンテコAIミーム風キャラを捕獲するタイミングアクション+図鑑コレクションゲーム。

- **プレイURL**: https://nrtkumi.github.io/games/aura-catch/
- **対応環境**: スマホ(縦画面・タッチ操作)ファースト。PCのマウス操作にも対応。依存ライブラリなしの単一HTML。

## 遊び方

1. 「キャッチ開始」をタップ
2. 画面中央にキャラが登場し、外側から白いリングが縮んでくる
3. リングがキャラのまわりの点線(ターゲット円)に重なった瞬間にタップ!
   - **金色に光った瞬間**にタップできれば「ガチ神」パーフェクト(オーラ2倍)
   - 緑のうちにタップでもキャッチ成功
   - 早すぎ・遅すぎはキャラに逃げられてハートを1つ失う
4. ハート3つがなくなったらゲームオーバー。キャッチするたびリングが速くなる
5. 連続キャッチでコンボボーナス。レアキャラほどオーラが大きい

### レアリティ

| レアリティ | オーラ倍率 | 体数 |
|---|---|---|
| ふつう | ×1 | 12 |
| レア | ×2 | 6 |
| エグレア | ×4 | 3 |
| シグマ | ×9 | 2 |
| ブレインロット級 | ×25 | 1 |

捕まえたキャラは「ブレロ図鑑」(全24体)に登録される。ベストオーラと図鑑はlocalStorageに保存され、コンプ率を上げていくのが長期目標。

## 作成の背景(市場調査の要点)

2026年6月時点の調査をもとに、日本のトレンドとアルファ世代トレンドを企画に反映した。

- **「イタリアンブレインロット」が日本の小学生に大流行**: AI生成のヘンテコキャラを人工音声で紹介するミームが2025年の小学生年間トレンド入り([週刊アスキー](https://weekly.ascii.jp/elem/000/004/357/4357615/)、[小学館AD POCKET](https://adpocket.shogakukan.co.jp/adnews/17469/))。Robloxの『Steal a Brainrot』は同時接続2,500万人超([こどもとIT](https://edu.watch.impress.co.jp/docs/report/2078881.html))→ **「ヘンテコキャラを集める」体験**をコアに据えた。キャラは絵文字の組み合わせ+オリジナルのノンセンス名24体で、既存ミームの名前・デザインは使用していない
- **アルファ世代スラング**: 「オーラ」「シグマ」「NPC」などゲームチャット発のスラングが定着([Classpop](https://www.classpop.com/magazine/gen-alpha-slang)、[SheKnows](https://www.sheknows.com/parenting/slideshow/1234883077/teen-slang-2026/))→ スコアを「オーラ」と呼び、判定演出に「ガチ神」「NPCムーブ…」などのスラングを採用
- **ハイブリッドカジュアル化**: 5〜10秒で理解できる単一メカニクス+収集・進行のメタレイヤーという構成が2026年の主流([Game Growth Advisor](https://gamegrowthadvisor.com/blog/2026-04-16-hybrid-casual-game-design-strategy-2026/)、[Udonis](https://www.blog.udonis.co/mobile-marketing/mobile-games/casual-games))→ タイミングタップ1メカニクス+図鑑コンプのメタレイヤー
- **日本のZ世代は「共感・ミーム」消費**: 「日常ミーム」やキャラへの愛着(ぬい活)がキーワード([SHIBUYA109 lab. トレンド予測2026](https://www.shibuya109.co.jp/shibuya109lab/reports/shibuya109-lab-%E3%83%88%E3%83%AC%E3%83%B3%E3%83%89%E4%BA%88%E6%B8%AC2026/)、[kanau-lab](https://kanau-lab.co.jp/blog/7553))→ 全キャラに一言コメント(「あんこは飲み物」等)を付け、図鑑を眺めて楽しめるようにした

サイト内の既存ゲーム(hakai=ブロック崩しローグライト、suika-fever=マージ系)とジャンルが被らないよう、タイミングアクション+収集系を選んだ。

## 技術構成

- 依存ライブラリなしの単一HTML(canvas + Pointer Events)
- フォントは既存ページと統一(Bungee + Zen Kaku Gothic New + Space Mono)
- 効果音はWeb Audioでリアルタイム合成。キャラ登場時の「意味不明ボイス」もオシレーターで生成
- **iOS消音スイッチ対策**: 初回タップ時に無音WAV(JSでArrayBuffer生成→Blob URL)を`<audio>`要素でループ再生してオーディオセッションを再生カテゴリへ切り替え、`AudioContext.resume()`のPromise完了を待ってアンロック判定、`onstatechange`で中断から自動復帰(games/hakai/と同方式)
- メインループ(rAF)はスタート画面の時点から回し、ループが触る状態は全て宣言時に初期化(初回フレーム例外でループが死ぬ事故の再発防止)
- ベストスコアと図鑑はlocalStorage(try/catchでフォールバック)
- テスト用に読み取り専用フック`window.__AURA__`を公開(state/phase/リング比率など)

## 実施したテスト

1. **構文チェック**: インラインscriptを抽出して `node --check` → OK
2. **ヘッドレスロジックテスト**: DOM/canvas/localStorageをスタブしてNode(vm)でゲームループを駆動。スタート画面でのループ生存→開始→パーフェクト帯でのタップ捕獲×3→放置で3ミス→ゲームオーバー→記録保存→図鑑表示(24枚)→リトライ、まで全16項目PASS
3. **実ブラウザE2E**: puppeteer-core + ローカルChrome、モバイルビューポート(390×844、`hasTouch: true`)+タッチイベントで実プレイ。pageerror/consoleエラーゼロ、canvas描画ピクセルあり、タッチで3体捕獲、ゲームオーバー、localStorage永続化、図鑑24枚、リトライまで全16項目PASS。スクリーンショットで見た目も目視確認

### 開発で得た教訓

- ヘッドレステストで「時間からタイミングを推定してタップ」すると判定がずれて捕獲できない。ゲーム側に読み取り専用のテストフック(`window.__AURA__.ratio()`)を用意し、実際のリング比率を見てタップする方式にしたらヘッドレス・E2E両方が安定した
- 画面右上に固定UIボタン(ミュート)を置く場合、オーバーレイ画面のヘッダーと重なりやすい。オーバーレイ側に`padding-right`を確保して回避
- favicon未指定だとE2Eの「consoleエラーゼロ」チェックが404で落ちる。インラインSVG(data URL)のfaviconで解消

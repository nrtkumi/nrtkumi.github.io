# ねんぴょう — 歴史を、順番に。

人物・建物・できごと・映画・本…**Wikipedia / Wikidata の実データ**から引いた歴史の「札」を、**古いほうから順に**年表へ差し込んでいくタイムライン・パズル。年は伏せられていて、推理して正しいすき間に入れる。正しく並べ続けるほど年表が伸び、**「あと1枚」**を狙う中毒ループにハマる。

- **プレイ:** https://nrtkumi.github.io/games/nenpyo/
- **対応:** スマートフォン(タッチ)推奨 / PC(←→キーでカーソル移動・Enterで配置)も可。縦画面・1人用・単一HTML。
- **通信:** 必要(Wikimedia Commons の画像と日本語版Wikipedia REST の要約をブラウザから直接取得)。

---

## 遊び方

1. 画面下に**札**が1枚出る。そこには名前・分類(朱印)・写真があるが、**年代は「？？？」で伏せられている**。
2. 上の**年表**(古い ← → 新しい)で、その札が入ると思う**すき間(▼)をタップ**する。
3. 位置が正しければ年表に差し込まれ、年号が判明して**連鎖(コンボ)**が伸びる。
4. 間違えると正しい位置に答えが置かれ(その札に「誤」印)、**残機(朱印)を1つ失う**。残機が尽きたら終了。
5. 正解数=年表の長さがスコア。自己ベストは端末に保存される。

### ルール・要素

| 要素 | 内容 |
|---|---|
| 残機 🟥 | 朱印3つでスタート。誤配置で1つ消費 |
| 残機回収 | 12枚正解ごとに残機+1(最大3) |
| 連鎖 | 連続正解でコンボ。途切れると0に。最長連鎖がリザルトに残る |
| 答え合わせ | 誤配置時、正しい年号と**Wikipediaの要約**(ライブ取得)をその場で表示して学べる |
| 分類の朱印 | 人=人物 / 築=建物・場所 / 戦=できごと / 書=書物 / 映=映画 / 遊=ゲーム / 音=音楽 / 船=乗り物 |
| シェア | リザルトから正解数を X / 共有シートへ |

**コツ:** 写真と分類が大きなヒント。肖像画の服装・建物の様式・本や映画の雰囲気から年代を推理する。紀元前(前◯◯年)の札は年表のいちばん左へ。

## 作成の背景

「Wikipediaの公開APIで遊ぶ」というお題に対し、サイトには既に *リンクを辿る* [のりかえ百科](../norikae/)と *閲覧数を当てる* [ペディアバトル](../pedia-battle/)があったため、**まったく別のメカニクス=「年代順に並べる」**を採用した。

- **下敷きにした実証済みのバイラル作:** 本作は Wikidata 製の時系列カードを年表に挿し込む人気ゲーム **Wikitrivia**(tomjwatson 作, https://wikitrivia.tomjwatson.com/)のコアメカニクスを、**スマホのタップ操作・日本語データ・残機制**に再設計したもの。Wikitrivia は「1枚ずつ伸びる年表」と「終盤の1ミスで全部終わる緊張感」で大きく拡散した。
- **なぜ中毒性があるか:** ①年表が伸びるほど挿し込めるすき間が増え、自然に難易度が上がる ②残機と連鎖が「あと1枚だけ」を誘発するハイスコア更新ループ(『2048』や Higher/Lower 系と同じ快感)③誤答時に出典(Wikipedia要約)が出るので「へぇ」という**学びの報酬**が毎回ある。
- **クイズではなく素材としての実データ:** 年代・写真・分類はすべて実在の項目に紐づく本物。プレイするほど歴史の前後関係が体に入る。

## 技術構成

- **単一HTML**(インラインCSS/JS、ビルド不要)。ゲーム本体は約700行。
- **データ源(ビルド時に厳選):** [Wikidata Query Service (SPARQL)](https://query.wikidata.org/) で「日本語版Wikipedia記事がある/画像がある/被リンク言語数(sitelinks)が多い=有名」な項目だけを8カテゴリ分取得し、年代・名前・分類・画像URLに整形して `data.json`(565件・約90KB)に固めた。SPARQLは `Access-Control-Allow-Origin: *` でCORS対応だが1クエリ30〜40秒かかるため、実行時ではなく**事前生成**している。
  - 年代の出典プロパティ: 人物=生年(P569)、映画/書籍/音楽=公開・出版日(P577)、建造物=建設(P571)、戦争=開始(P580)。
- **実行時のライブAPI:** 札の写真は [Wikimedia Commons](https://commons.wikimedia.org/) の `Special:FilePath/...?width=` を直接 `<img>` で読む(縮小サムネをCDN生成)。誤答時の解説は [日本語版Wikipedia REST API](https://ja.wikipedia.org/api/rest_v1/) の `page/summary` をライブfetch(オフラインでも年号表示にフォールバック)。
- **デザインコンセプト:** 「博物館の年表アーカイブ / 古文書」。生成り色の和紙テクスチャ(SVGノイズ)、明朝体(Shippori Mincho B1)、分類と正誤を**朱の落款印(漢字一字)**で表現。AI感の出る紫グラデ・グラスモーフィズムを排した。
- **iOS/スマホ対策:** 消音スイッチ対策の無音WAVループ + `AudioContext.resume()` 待ち + `interrupted` 自動復帰、Web Audioによる効果音リアルタイム合成(外部音源なし)、`viewport-fit=cover`+セーフエリア、Pointer Events統一(`click`非依存)、ダブルタップズーム抑止3点セット、`max-width`縦長カラムでPCワイド対策・キーボード操作対応。

## 実施したテストと教訓

- **構文チェック:** インラインscriptを抽出し `node --check`。
- **データ検証:** 取得した665件すべての画像サムネ(`width=400`)を実際にHTTP取得して**生きているURLだけ**を採用(565件)。→ **教訓:** Commonsへ高並列(24)でHEADすると429スロットリングで大量に偽の失敗が出る。並列6+リトライに落として解決。
- **実ブラウザE2E(puppeteer-core + ローカルChrome, iPhoneビューポート/タッチ):** ローカルHTTPサーバ経由で、データ読込→開始→**15連続正解(スコア15・コンボ15)**→誤配置で残機を失い**ゲームオーバー**→リトライでリセット、までを実タッチ/Pointerで走破。`pageerror` 0件、`/games/nenpyo/` のリクエスト失敗0件、Commons画像のライブロード(330×452)も確認。残る404は `favicon.ico` のみだったため inline SVG favicon を追加して解消。
- **教訓:** 年代順の判定は同年(タイ)を考慮し、`年<対象 の件数 lo`〜`年≦対象 の件数 hi` の範囲 `[lo,hi]` を正解スロットとした(同じ年の札はどちら側でも正解)。

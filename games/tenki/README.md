# せかい天気くらべ — 今いちばん○○な街はどこ?

**今この瞬間**、世界でいちばん「暑い・寒い・風が強い・じめじめした」街はどこ? — 4つの都市から当てる、**リアルタイム気象**の勝ち残りクイズ。気温・風速・湿度はすべて **Open-Meteo の今の観測値**、国旗は **flagcdn** の本物。季節や半球(南半球は逆!)を読みながら連勝を伸ばそう。

- **プレイ:** https://nrtkumi.github.io/games/tenki/
- **対応:** スマートフォン(タッチ)推奨 / PC(マウス・数字キー1〜4)も可。縦画面・1人用・単一HTML。

---

## 遊び方

1. 「はじめる」を押すと、世界の主要都市の**今の天気**を取得します。
2. お題(下表)に当てはまる街を、**4つから1つタップ**。
3. 正解すると **連勝(れんしょう)** がのびて次のお題へ。**1回でも外したら終了**。
4. 5連勝ごとにお祝い演出。最高連勝は端末に記録されます。
5. 右下の「↻ 最新に更新」で観測値を取り直せます(観測時刻を表示)。

### お題(ランダム出題)

| お題 | 判定 | 指標(Open-Meteo) | 単位 |
|---|---|---|---|
| 🔥 あつい | いちばん高い | `temperature_2m` | ℃ |
| ❄️ さむい | いちばん低い | `temperature_2m` | ℃ |
| 🌬️ 風がつよい | いちばん速い | `wind_speed_10m` | km/h |
| 💧 じめじめ | いちばん高い | `relative_humidity_2m` | % |

収録は世界の主要都市58。**コツ:** 緯度・季節・半球(北が夏なら南は冬)、砂漠/海洋性気候などを考えると当たりやすい。当たらなくても「シドニーは今9℃!?」という驚き＝学びになります。

## 作成の背景

「ブラウザから使える無料の公開API」を主役にしたゲームの第2弾として、**リアルタイム性**を遊びの核に据えました。

- **メカニクス選定:** 「4つからいちばんを選ぶ→正解で連勝、外したら即終了」という**勝ち残り(ハイスコア更新)ループ**は、結果が予測不能なほど **報酬予測誤差(ドーパミン)** が大きく、「あと1問」の **near-miss** が継続意欲を高めることが知られています(Schultz 1997 ほか / [可変報酬の心理](https://neurolaunch.com/variable-reward-psychology/)、[near-miss の神経科学](https://pmc.ncbi.nlm.nih.gov/articles/PMC3077261/))。操作は1タップに絞り即時フィードバック([CHI'22: hyper-casual loop](https://dl.acm.org/doi/fullHtml/10.1145/3491101.3519837))。
- **リアルタイムの驚き:** 毎回データが変わるので**手が止まらない**。実際の気象から「南半球は今が冬」「砂漠は夜でも暑い」といった気づきが生まれる教育的副産物も。
- **データ源:** キー不要・CORS対応で**全都市を1リクエストで取得**できる **[Open-Meteo](https://open-meteo.com/)**。国旗は CORS 対応の **[flagcdn.com](https://flagcdn.com/)**。
- **片手・縦画面設計:** 4枚のカードを親指の届く範囲に配置([thumb zone](https://www.smashingmagazine.com/2020/02/design-mobile-apps-one-hand-usage/))。

> 既存ゲームとAPI・メカニクスが被らないことを確認(天気API・「4択でいちばんを選ぶ」形式はサイト初。前作 kunikurabe は World Bank の2択ハイ&ローで別物)。

## 技術構成

- **単一HTML**(インラインCSS/JS、ビルド不要)。都市リスト(名前・緯度経度・国コード)は内蔵、天気は**起動時に1リクエストで全都市分を取得**(`current=temperature_2m,relative_humidity_2m,wind_speed_10m,weather_code`、`timezone=auto`)。
- **堅牢性:** タイトル表示中に裏で先読み。取得失敗時は無言で壊さず「再試行」画面を表示。同点で割れないよう、出題時は対象指標の値がユニークな4都市を選定。
- **国旗:** `https://flagcdn.com/w160/{iso2}.png`、読み込み失敗時は非表示にフォールバック。天気アイコンは WMO weather_code から記号化。
- **デザインコンセプト:** 「晴れた空のグアッシュ＋切り紙のお天気カード」— 水色の空・流れる雲・太陽、丸ゴシック(Zen Maru Gothic / Baloo 2 / Kosugi Maru)。AI的な紫グラデ/グラスモーフィズムは不使用。背景の雲は canvas でゆっくり流れる。
- **iOS対応:** Web Audio はマナーモードで無音化するため、初回タップで無音WAVループ + `AudioContext.resume()` 待ち + `onstatechange` 復帰。効果音は Web Audio 合成。ダブルタップズーム無効化3点セット、Pointer Events 統一(`click`非依存)、セーフエリア対応、PCワイド対策(縦カラム中央寄せ)。
- **永続化:** localStorage に最高連勝(try/catch)。

## 実施したテストと教訓

1. **構文チェック:** インラインJSを抽出し `node --check` → OK。
2. **ヘッドレスロジックテスト:** 都市データの整合、**300ラウンドのランダム出題**で「正解＝指標の最大/最小」かつ「4都市の値が同点で割れない」ことを検証 → PASS。
3. **実ブラウザE2E(puppeteer-core + Chrome, モバイルVP `hasTouch`):** 起動→「はじめる」→ **Open-Meteo のライブ取得成功(4カード描画)** を確認。出題ごとにカードを公開し、**「正解カードが実データの極値と一致(mismatch 0)」を3ラウンド検証**、**国旗の実画像ロード(naturalWidth=160)**。`pageerror`/コンソールエラーは favicon 以外ゼロ。スクリーンショット目視OK。

**教訓:** Open-Meteo は**複数地点をカンマ区切りで1リクエスト**にまとめられ、全都市分の現況をまとめて取れる(都市ごとに叩かない)。リアルタイムAPIは「毎回違う」こと自体が中毒性の燃料になる。`sips --out` は出力先ディレクトリが無いと**その名前のファイルを作ってしまう**ので、`assets/` を先に作ること。

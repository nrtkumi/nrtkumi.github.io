# クロヌリ — 黒塗り百科事典

黒塗り(検閲)にされた百科事典の項目から、**正体を暴く**4択クイズ。手がかりが少ない段階で当てるほど高得点。連続正解で倍率、㊙(ライフ)が尽きるまで延々と遊べる、**Wikipedia実データの中毒性クイズ**。

- プレイ: https://nrtkumi.github.io/games/kuronuri/
- 対応: スマホ(タッチ)/PC。縦画面・単一HTML。オンライン必須(Wikipedia APIを参照)。

## 遊び方

1. 黒塗りの「機密ファイル」(=ある記事の冒頭文)が出る。記事名と読み・固有名は墨で塗りつぶされている。
2. 残った手がかりから、**4つの候補**の中で正体を選ぶ。
3. 文は段階的に**解禁**される: 最初は一文だけ → 全文 → 添付写真。**早い(=手がかりが少ない)段階で当てるほど高得点**。
4. 連続正解で**倍率アップ**。間違い・時間切れで㊙が1減り、0でゲームオーバー。
5. 進むほど機密度ランク(D→SSS)が上がり、よりマイナーな項目が出る。

## なぜ中毒性があるか

- **「あと一問」ループ**: 1問が数秒、即リトライ、ハイスコア更新。
- **リスク&リターン**: 手がかりが少ないうちに賭けて当てれば高得点、待てば易しいが低得点。
- **実データの新鮮さ**: 出題は毎日変わる「実際によく読まれている記事」。時事ネタも歴史も人物も出るので飽きにくい。
- **巧妙な誤答**: ダミー選択肢は本物と意味的に近い記事(例: 黒田孝高 → 豊臣秀吉・黒田長政・軍師官兵衛)なので、適当では当たらない。

## 作成の背景

当サイトには既に Wikipedia/Wikidata を使ったゲームが3本あるが、いずれもコアループが異なる:
- **のりかえ百科**(norikae): リンクを辿るWikiRace
- **ペディアバトル**(pedia-battle): 閲覧数の Higher/Lower
- **ねんぴょう**(nenpyo): 年代順に並べるタイムライン

本作は「**黒塗りの記述文から記事名を当てる**」という未使用のメカニクスで、これらと被らない。お題は **Wikimedia pageviews API の人気記事**から採り(認知度が高く当てやすい)、誤答は **action API の `morelike` 検索**で意味的に近い記事を集めている。本文・読み・写真は **REST `page/summary`** から取得。すべて CORS 対応(`Access-Control-Allow-Origin: *`)でキー不要。

- pageviews(人気記事): https://wikimedia.org/api/rest_v1/metrics/pageviews/top/ja.wikipedia/all-access/{YYYY}/{MM}/{DD}
- 要約: https://ja.wikipedia.org/api/rest_v1/page/summary/{title}
- 類似記事(誤答): https://ja.wikipedia.org/w/api.php?action=query&list=search&srsearch=morelike:{title}&origin=*

## デザインコンセプト

**黒塗りの機密ファイル(検閲済み公文書)**。マニラ紙のファイル、紙クリップ、ルーズリーフの罫線、㊙の朱印、タイプライター体のファイル番号。墨の黒塗りバーで記事名を隠し、正解すると「解禁」スタンプと添付写真(最初はブラー)が公開される。タイプライター体(Special Elite)＋明朝(Shippori Mincho)。生成AI的な紫グラデ・ガラス質感・ネオンは排した。

## 技術構成

- 単一HTML・依存ライブラリなし。**DOMベース**(機密文書の表現に最適)＋ Web Audio のリアルタイム効果音。
- **黒塗り処理**: 記事名・読み・各トークン(姓/名など、2字以上)・冒頭の括弧(読み/外国語名)を本文中から検出して墨バーに置換。Wikipediaが人名を「姓 名」と空白入りで表記するケースに対応するため、**空白・中黒を無視したマッチング**で名前の漏れを防ぐ。
- **先読み**: 次の問題を裏で常に1〜2問プリフェッチし、回答後すぐ次へ出せるようにして「延々と」遊べる体感を確保。`AbortController` でタイムアウト、失敗問題は静かにスキップ。
- iOS消音スイッチ対策・ダブルタップズーム無効化・セーフエリア対応・PCワイド画面の縦カラム制限。得点/最高記録は localStorage。

## 実施したテスト

- インラインJSを抽出し `node --check`(構文OK)。
- 実ブラウザE2E(puppeteer-core + ローカルChrome, iPhoneビューポート/タッチ, **実際のWikipedia APIに接続**): 出題ロード→黒塗り表示の確認→正解で加点(倍率上昇)→誤答で㊙減少→ゲームオーバー、を通し、pageerror ゼロ(faviconの404のみ)を確認。
- **名前漏れの自動検査**: 連続8問について「記事名(空白・中黒除去)が本文に露出していないか」を機械チェックし、漏れ0件を確認(人名の「姓 名」表記や外国人名のミドルネーム形に対応)。
- `file://` ではなくローカルHTTPサーバ経由で実施(CORSのOrigin要件のため)。

## 教訓

- 日本語Wikipediaの要約は人名を「**姓 名**」と空白入りで書くことが多く、記事タイトル(空白なし)とそのままでは一致せず**答えが本文に漏れる**。空白・中黒を無視したマッチングとトークン単位の黒塗りで対処した。漏れ検査をE2Eに入れて初めて気づけた。
- `page/related` は現在403。誤答は action API の `morelike` 検索が良質(意味的に近い＝難しく面白い誤答)。取得できない時は人気記事プールからフォールバック。

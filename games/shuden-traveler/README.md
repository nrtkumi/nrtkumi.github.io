# 終電トラベラー — 終電が尽きるまで、どこまで行ける？

夜18:00の東京駅。**本物の発車時刻表**だけを頼りに列車を乗り継ぎ、**終電が尽きる前に日本のどこまで遠く行けるか**に挑む push-your-luck(運試し)ローグライト。新幹線は一気に距離を稼げるが、遠くへ跳ぶほど乗り継げる終電は少なくなる——その駆け引きがコア。

- **プレイ:** https://nrtkumi.github.io/games/shuden-traveler/
- **対応:** スマートフォン(タッチ)推奨 / PC も可。縦画面・1人用・単一HTML。

---

## 遊び方

1. 東京駅18:00からスタート。下の**発車案内**に、いま乗れる列車(本物の時刻・路線・行き先・距離)が並びます。
2. 列車をタップで乗車。**その列車の発車時刻まで時計が進み**、到着駅へ移動。走破距離が加算されます。
3. 到着駅で次の発車案内が開く。これを繰り返して旅を伸ばす。
4. **乗り継げる終電がなくなったら旅は終了**(深夜2時で打ち切り)。スコアは累計の**走破距離(km)**です。

### コツ・駆け引き

- 🚄 **新幹線**は1本で数百kmを稼げるが、遠方は終電が早く、跳んだ先で**立ち往生**しやすい。
- 在来線でこまめに繋ぐと乗車本数は伸びるが、距離は稼ぎにくい。
- 待ち時間が**90分を超える接続は「もう終電がない」扱い**で表示されません。リアルな終電事情がそのまま難易度になります。

### 評価(累計走破距離)

| 走破 | 称号 |
|---|---|
| 1500km〜 | 🏆 日本縦断の鉄人 |
| 900km〜 | 🚄 終電マスター |
| 450km〜 | 🌃 夜行の達人 |
| 180km〜 | 🚃 そこそこ遠征 |

自己ベスト走破距離は端末に保存されます。

## 作成の背景

**「Transit API で遊ぶ」**というお題に対し、近年人気の**push-your-luck(運試し)/ローグライト**を、実在の時刻表ネットワークに乗せた「終電トリップ」として設計しました。

- **なぜ面白い/中毒性:** 「あの新幹線に乗れば一気に遠くへ。でも降りた先で終電が無かったら…」という**引き際の判断**が毎ターン発生。地理の知識と運が噛み合い、リアルな終電が自然な制限になります。日本地図上に旅の軌跡が伸びていく視覚的快感も核。
- **押し引きの設計:** サイトの過去作の教訓(加算式だと即降りが最適化して押し引きが死ぬ)を踏まえ、本作は「**距離(報酬)と終電(時間資源)のトレードオフ**」で内点最適を作っています。遠くへ跳ぶほど時間を多く消費し、乗り継ぎ候補が痩せる。
- **データ源:** [Transit API](https://api.transit.ls8h.com) の `GET /api/v1/locations/suggest`(駅名→同一駅の複数路線=のりばを座標近傍で集約、行き先名の解決)と `GET /api/v1/stations/{id}/departures`(各のりばの実発車時刻表)。複数フィードの発車標をマージして一つの「ターミナル発車案内」を再現。読み取り専用・認証不要・CORS開放。

## 技術構成

- **単一HTML**(インラインCSS/JS、ビルド不要)。`<canvas>` で日本地図(等距円筒図法・主要都市ラベル)に走行ログを描画。
- **デザインコンセプト:** **深夜のネオン**。藍色の夜空+星、シアン/マゼンタの発光、`Zen Kaku Gothic New`。地図を主役に据えた「夜の旅」の雰囲気。
- **時計モデル:** 終電はサービス日0:00からの秒数(86400超あり)で表現。発車案内は現在時刻以降かつ**待ち90分以内**の便だけを表示し、深夜の「終電が来ない」状況を再現。新幹線は路線名/列車名(のぞみ・はやぶさ等)で判定し、移動時間と表示を区別。
- canvas は iOS Safari 対策で `flex:1 1 0; min-height:0` とし、毎フレーム `setTransform(dpr,…)` で変換を再設定。
- 効果音は Web Audio 合成、iOS 消音スイッチ対策(無音WAVループ + resume 待ち + interrupted 復帰)。ベストは localStorage。

## 実施したテスト

- インラインJSの構文チェック(`node --check`)。
- 実機相当の puppeteer E2E(iPhone ビューポート・タッチ、ローカルHTTPサーバ経由):複数戦略(最遠優先/最速優先)で東京から乗り継ぎ、時計が現実的に進行(例:東京→新大阪 20:11→熊本 23:01→博多→終電)し、終着・走破距離・最遠到達・地図描画が機能、`pageerror`/console エラーが出ないことを確認。
- 開発で得た教訓:`departures` は夜遅い時刻で問い合わせると**翌朝のサービス便(大きな秒数)も返す**ため、`現在時刻 ≤ 発車 ≤ 現在時刻+90分`で絞らないと時計が非現実的に跳ぶ。新幹線は路線名に「新幹線」を含まない(のぞみ等の列車名)ことがあり、列車名でも判定する必要がある。

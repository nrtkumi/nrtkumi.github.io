# オトアメ 音雨 (OTOAME) — 雨を奏でるリズムゲーム

4つの水路に**雨粒**が落ちてくる。水面のラインに**重なる瞬間**に、その列を**タップ**して弾く。タップした音だけが**メロディ**になり、コンボが続くほど雨は**光の音色**に変わっていく。テンポと密度が上がり続ける、降りやまない雨のリズムタップゲーム。

- **プレイURL**: https://nrtkumi.github.io/games/otoame/
- **対応環境**: スマホ(iOS Safari / Android Chrome)・PCブラウザ。縦画面・タッチ操作。PCはキー **D F J K**(左から各列)でも演奏可。

## 遊び方

1. 「奏でる」をタップ。
2. 雨粒が上から落ちてくる。**水面のライン(各列の○印)に粒が重なる瞬間に、その列をタップ**して弾く。
3. ジャストで弾くほど **PERFECT**(高得点・水しぶき大)。少しズレると **GOOD**。
4. 弾けずに水面を通り過ぎると **MISS** で体力(上部バー)が減り、コンボが切れる。**体力が尽きると雨やどり(ゲームオーバー)**。
5. コンボが伸びるほど倍率が上がり(×2〜×8)、音色が高く明るくなり、×4以上で **FEVER** 演出に。

### スコアと判定

| 判定 | 効果 |
|------|------|
| PERFECT(ジャスト) | 100×倍率。体力小回復、水しぶき大、音が澄む |
| GOOD(ややズレ) | 55×倍率。体力小回復 |
| MISS(取り逃し) | 体力 −16、コンボ・倍率リセット |
| コンボ | 10ごとに倍率+1(最大×8)。音程が1段上がる |

終了時は精度から **神 / S / A / B / C / D** のグレードが付く。最高スコアは localStorage に保存。

## デザインコンセプト

**夜の雨と水面**。藍緑(ティール)の水面に白い雨粒が落ち、しずくが弾けて波紋が広がる。背景には常時、細い雨筋が降る。コンボが伸びると水面と雨が金色〜光に染まる。フォントは明朝の Shippori Mincho。暖色の祭り(ヤグラ)・昼空(カミヒコーキ)に対して、冷たく静かな夜の雨で色調を分けた。

## 作成の背景

レーン/タイル式のリズムタップは、Piano Tiles 系として世界的に巨大なジャンル。旗艦の『Magic Tiles 3』は2026年5月時点で米App Store ミュージックカテゴリ1位、77.4万件で4.6/5、累計約8.1億DLとされる。コアは「指1本で落ちてくるタイルをタップ、白(空きレーン)を踏むと即ゲームオーバー、曲レベルごとに速度が上がる」("1. Tap black tiles 2. Avoid white tiles 3. Speed up each song level")。中毒性は**タップごとに鳴るピアノ音・目に見えるコンボ倍率(x990, PERFECT x26 など)・易しい入り口と高い天井**から来る。

本作は「即死」ではなく**体力ゲージ式**にして取っ付きやすくしつつ、**タップした音だけがメロディを形作る**よう設計した(裏で軽いキック/ベースのグルーヴが流れ、タップ音はペンタトニック音階で常に調和する=多少外しても曲が破綻しない)。テンポと密度を時間で上げ続け、1曲固定ではなくスコアチェイス型にしてある。1プレイが短く、スコアとグレードで自慢でき、人に勧めやすい。

## 技術構成

- **単一HTMLファイル**。canvas 2D で描画。外部ライブラリ・画像なし(雨粒・波紋・水しぶき・水面のゆらぎはすべてコード生成)。
- フォントのみ Google Fonts(Shippori Mincho / Zen Maru Gothic)を CDN 読み込み。
- **音楽は Web Audio でリアルタイム合成**。各列はペンタトニック音階の音を担当し、コンボで音程が1オクターブまで上昇。タップ音(オルゴール風トライアングル＋倍音)＋裏のキック/ベースのグルーヴが拍に合わせて鳴る。マスターゲイン経由でミックス。
- **タイミングモデル**: 一定の拍グリッド(120BPM・8分音符)上に、難易度カーブで密度を上げながらノートを先行スポーン。落下は「目標時刻に水面ラインへ到達する」よう逆算して配置。判定はジャスト(±0.075s)/GOOD(±0.15s)で、通過から猶予(0.17s)を超えるとMISS。
- **iOS消音スイッチ対策**: 無音WAVループ＋`resume()` 完了待ち＋`onstatechange` 復帰。
- ダブルタップズーム無効化、入力は Pointer Events と D/F/J/K キー。タップ位置→列をマッピング(`click` 非依存)。
- メインループはスタート画面から回り、状態は宣言時に全初期化。記録は localStorage(try/catch)。

## テスト

1. **構文チェック**: `node --check`。
2. **実ブラウザE2E**(puppeteer-core + ローカルChrome、ローカルHTTPサーバ経由、モバイルビューポート `hasTouch`): `pageerror`/console エラーゼロ、canvas 描画ピクセル(色数160超)、各列タップでスコア加算・MISSで体力減・体力0でゲームオーバー画面とグレード表示まで遷移することを確認。
3. プレイ中のスクリーンショットで、4レーン・雨粒(しずく形)・水面のゆらぎ・波紋・判定表示が正しく描画されることを目視確認。

### 教訓

- リズムゲームは「外しても曲が破綻しない」設計(裏グルーヴ＋ペンタトニック)にすると、初見でも気持ちよく演奏でき敷居が下がる。
- ノートの落下は「現在時刻からの逆算」で位置を決めると、フレームレートに依らず判定ラインに正確に届く。

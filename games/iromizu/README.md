# 色水 — IROMIZU 藍染め工房

染桶に積もった**色水**を注ぎ分け、それぞれの桶を**一色**にそろえる。反物の注文をこなすほど色が増え、道具を選んで挑む**水ソート・パズル × ローグライト**。

- プレイ: https://nrtkumi.github.io/games/iromizu/
- 対応: スマホ(タッチ)/PC(数字キーで桶選択、Uで戻す、Escで選択解除)。縦画面・単一HTML。

## 遊び方

1. 桶をタップで**持ち上げ**、もう一つの桶をタップで**注ぐ**。
2. 注げるのは「注ぎ口の色が同じ」か「空の桶」で、空きがある時だけ。**同じ色は続けて流れる**。
3. すべての桶を**一色だけ(または空)**にできれば、その注文(反物)は完成。
4. 少ない手数で仕上げると「手際よし」ボーナス。連続で仕上げると連反ボーナス。
5. 手詰まりになったら道具で打開。道具も尽きて打つ手がなければ仕事じまい(ゲームオーバー)。

### 道具(ローグライトの強化)

| 道具 | 効果 |
|------|------|
| 予備桶 | 空の染桶を1つ増やす(以降ずっと) |
| 漉し布 | どれか1段を取り除く札 +1 |
| 色見本 | 手を戻す(アンドゥ)札 +2 |
| 一掬い | 色を無視して1段だけ注ぐ札 +1 |
| 早見え | 今すぐ有効な一手を光らせる札 +2 |
| 大桶 | 全桶の容量+1(より多く積める) |

色は **日本の伝統色**(藍・紅・萌黄・山吹・紫・茶・鼠)。各色に小さな**紋**(点・線・×・輪 など)を付け、色だけに頼らず識別できるよう色覚に配慮した。

## 作成の背景

2026年の市場調査(deep-research で出典付き検証)で、**ソート系(水ソート/Coin Sort/Magic Sort!)**がハイブリッドカジュアルのトップIAPを占める持続トレンドだと判明。AppMagic の四半期レポートでは Sort 系が前年比 5.6 倍、新規ヒットは「過去のバイラルメカニクス(水ソート等)の再生・再結合」から生まれていると結論づけられている。当サイトに水ソートは無かったため、これを**染物屋テーマ + ローグライトの道具ドラフト + 色覚配慮の紋**で差別化して制作した。

- ハイブリッドカジュアルのパズル/ソート優勢: https://appmagic.rocks/blog/hybridcasual-q1-2025/?hl=en
- 旧バイラルメカニクスの再生(Coin Sort 等): https://appmagic.rocks/blog/q3hybrid2025

## デザインコンセプト

**藍染め工房(染物屋)**。麻・生成りの作業場、木目と箍(たが)の染桶、藍を主役に紅・萌黄・山吹などの染料、上部に干された染め布。水面は毎フレーム揺れ、注ぐと弧を描いて流れ落ちる。フォントは見出しに Zen Old Mincho。生成AI的な紫グラデ・ガラス質感・ネオンは排した。

## 技術構成

- 単一HTML・依存ライブラリなし。canvas 2D で染桶・色水・注ぎアニメを描画。
- **解ける盤面の保証**: 各色を容器いっぱいに満たした完成形からトークンをランダム配分し、DFS(visited 付き・ノード上限つき)ソルバで可解性を検証してから出題。手詰まり(合法手なし)も検出して道具で打開できる。
- iOS消音スイッチ対策(無音WAVループ＋AudioContext復帰)、ダブルタップズーム無効化、セーフエリア対応、PCワイド画面で縦カラムに制限。効果音はWeb Audioのリアルタイム合成(注ぐ高さでピッチ変化)。
- 得点/最高記録は localStorage に保存。

## 実施したテスト

- インラインJSを抽出し `node --check`(構文OK)。
- 実ブラウザE2E(puppeteer-core + ローカルChrome, iPhoneビューポート/タッチ): 同梱ソルバで**3注文を実際に解ききり**、レベルクリア→道具ドラフト→次注文の進行、pageerror ゼロ、canvas描画、各画面のスクリーンショット目視を確認。`file://` ではなくローカルHTTPサーバ経由で実施。

## 教訓

- 水ソートは「ランダム配置が必ず解けるとは限らない」のが落とし穴。**完成形からの逆算 or 生成→ソルバ検証**が必須で、ソルバはそのまま手詰まり検出・ヒント・E2Eの自動クリアに再利用できた。
- 色だけの識別はアクセシビリティ上弱いので、各色に**紋**を重ねた。OGP・小画面でも色の取り違えが起きにくい。

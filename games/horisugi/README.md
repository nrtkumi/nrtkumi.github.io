# ホリスギ！爆熱ドリル (HORISUGI - OVERHEAT DRILL)

**押すと掘れる。押しすぎると爆発。** 長押しだけで地球を掘り進む、赤熱チキンレース採掘インクリメンタル。
熱いほど速く掘れて儲かるが、熱100%で爆発して戦利品をロスト。深部では地熱で冷却が追いつかなくなり、「あと一掘り」と「帰還」の駆け引きが牙を剥く。

- **プレイURL**: https://nrtkumi.github.io/games/horisugi/
- **対応環境**: スマホ(縦画面・ワンサム長押し)ファースト。PC(Space長押し / Enter・Rで帰還)対応。依存ライブラリなしの単一HTML。

## 遊び方

1. **画面を長押し** — ドリルが掘り進む。掘るほど熱ゲージが上昇
2. **赤熱ゾーン(熱78%〜)** — 掘削速度1.5倍・鉱物出現1.8倍、維持するとコンボ倍率が最大x2.5まで成長。ゾーンを外れるとコンボは即減衰
3. **熱100% = 爆発** — そのダイブで積んだ鉱石をロスト(保険カプセルで一部回収可)。指を離せば冷える
4. **深部は地熱が強い** — 「冷却限界深度」(点線で表示)より深くは、離しても熱が上がり続ける。爆発する前に**帰還**して換金
5. **ジオで強化** — ドリル刃/冷却ファン/耐熱合金/集鉱磁石/発進エレベーター/保険カプセル+取り外し不可の二択改造(爆熱炉⇔急冷システム、ギャンブルビット⇔堅実ビット)
6. **深度8,000mの地球のコア到達**でエンディング。以降は到達ごとに全鉱物価値+10%が累積する周回ボーナス

## 作成の背景(市場調査)

「中毒性」をコンセプトに、2025〜26年のトレンドを調査して企画:

- **採掘・インクリメンタル系が爆発的トレンド**: 一人開発の『A Game About Digging A Hole』が[100万本超](https://www.gamesmarket.global/viral-hit-a-game-about-digging-a-hole-tops-one-million-sales-82fed80996941736a1a54c31381ab20b/)、掘って売って強化する短いループが核。『[Digseum](https://store.steampowered.com/app/3361470/Digseum/)』(97%好評)や『[Nodebuster](https://mancunion.com/2025/05/08/nodebuster-review-a-short-and-sweet-incremental-game/)』など「短時間ドーパミン特化インクリメンタル」が一大潮流に。日本でも『[無限採掘場 気づいたら世界破砕](https://www.gamespark.jp/article/2026/04/24/165598.html)』が同接6千人を記録
- **ギャンブル心理の設計が再評価**: 『[CloverPit](https://www.pcgamesn.com/cloverpit/slot-machine-roguelike)』はBalatro以降のヒットとして、変動報酬・ニアミス・大当たり演出の快感を実証
- **ブラウザゲーム文脈**: Neal.funの『[Stimulation Clicker](https://en.wikipedia.org/wiki/Stimulation_Clicker)』がクリッカーの即時フィードバックで2025年に話題。[デイリーパズル/直感的操作のゲームがブラウザ市場の主役](https://dopabrain.com/portal/blog/en/best-browser-games-2026.html)
- **ハイパーカジュアルの定石**: [1〜3分セッション+メタ進行のハイブリッド化](https://www.blog.udonis.co/mobile-marketing/mobile-games/casual-games)

これらを「**長押し1操作のリアルタイムpush-your-luck**」として統合した。差別化の核は採掘ではなく**熱を賭ける駆け引き**:
赤熱ギリギリのサーフィン(変動報酬+ニアミス)、地層ごとに適正温度が反転する状況判断、深度による帰還タイマー(地熱)、爆発の全ロスト(損失回避)、コンボ・大当たり・発見の三段ドーパミン。

企画・実装の両段階でOpenAI Codex(gpt-5.5)によるレビューを実施し、「帰還をプレイヤーの決断にする」「赤熱維持が常に正解になるのを地層ルールで崩す」「コンボの安全圏ファーミング防止」「マルチタッチ/visibilitychange対応」「roundRectポリフィル」「リフレッシュレート非依存のパーティクル生成」など計18件超の指摘を反映した。

## 技術構成

- 依存ライブラリなしの単一HTML(canvas 2D + Web Audio)
- 効果音は全てWeb Audioでリアルタイム合成。ドリル音はノイズ→バンドパス+サブオシレータの常駐ノードで、熱に応じて周波数・音量が連続変化
- iOS消音スイッチ対策: 初回タップで無音WAVを`<audio>`ループ再生→`AudioContext.resume()`のPromise完了待ち→`interrupted`からの自動復帰
- ダブルタップズーム対策3点セット(`touch-action`、`dblclick`/`gesturestart`の抑止、350ms内連続`touchend`の抑止)。入力はPointer Events統一+pointerId管理で多指・中断に頑健
- 地形装飾は決定的ハッシュnoiseで生成(スクロールしても同じ場所に同じ小石)
- DPR変換は毎フレーム`ctx.setTransform`で再設定(コンテキスト喪失対策)。DOM画面表示中はcanvas描画を2fpsに抑制(電池対策)
- セーブはlocalStorage(try/catchフォールバック)
- OGP画像は実プレイをpuppeteerで自動操縦し赤熱91%の瞬間を撮影→タイトルロゴをHTML合成→1200x630

## 実施したテスト

1. **構文チェック**: インラインscript抽出→`node --check`
2. **ヘッドレス機能テスト(23項目)**: DOMスタブで実コードを駆動し、開始→掘削→収集→帰還換金→localStorage永続化→ショップ購入→爆発全ロスト→保険回収→5000フレーム安定性を検証
3. **経済シミュレーション**: bot方策で全進行を自動プレイし、初強化がダイブ1〜3、800m=4分、2,500m=12分、コア到達=慎重プレイで83分(赤熱サーフィンで短縮可)の曲線に調整。慎重プレイなら爆発0(死は常にプレイヤーの強欲)であることも確認
4. **実ブラウザE2E(11項目)**: puppeteer-core+ローカルChrome、モバイルビューポート+タッチイベントでローカルHTTP経由の実プレイ。pageerror/console.errorゼロ、canvas描画ピクセル、タッチでの開始/帰還/ショップ/爆発、スクリーンショット目視確認

## 開発で得た教訓

- `codex exec`はgitリポジトリ外のディレクトリを`-C`で指定すると信頼チェックで無限に停止する(`--skip-git-repo-check`が必要)
- Bungeeフォントは小文字グリフがないため「18m」が「18M」に見える。単位付き数値はSpace Mono等を使う
- 経済バランスはヘッドレスbotシミュレーションで一発検証できる(初版はダイブ4で全強化が買えてしまうインフレを事前に検出)

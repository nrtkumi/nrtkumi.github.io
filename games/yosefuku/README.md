# 寄せ福 — YOSEFUKU

縁起物の御札を回し、隣り合う札の「ご縁」(隣接シナジー)で福銭を稼ぐ、和風スロット・ローグライト。
回した後に行や列を **「寄せて」** 札を揃え直す一手が、この遊びの肝。

- **プレイ:** https://nrtkumi.github.io/games/yosefuku/
- **対応:** スマホ(縦・タッチ)ファースト。PC(マウス/キーボード)でも遊べる。
- **デザインコンセプト:** 「神社の縁日・千社札」。朱漆(しゅうるし)×金×生成りの漆器パレット、御札=漢字木札、和柄(麻の葉)の薄い地紋。フォントは見出しに Reggae One、本文に Shippori Mincho B1、札の漢字に Yuji Syuku。生成AIが作りがちな紫グラデ/グラスモーフィズムは排除。

## 遊び方

1. **回す** — 3×3の御札盤に、所持する縁起物がランダムに並ぶ(リール演出)。
2. **寄せる(本作の肝)** — 回したあと、**行や列をスワイプ**(またはガターの矢印)で1マスずつズラせる(ラップ・1回の回りにつき2手)。隣接シナジーや角ボーナスが揃うように札を寄せて、福銭を最大化する。
3. **収める** — その回りの福銭を財布に入れる。1ステージ(祭)は3回まで回せる。
4. **お供え** — 祭の終わりに「お供え(ノルマ)」を払う。足りればクリア、余りは次の御利益(ショップ)で使える。払えなければ願いはそこまで。
5. **御利益** — 祭のあいだに札を1枚授かる(または納めて入れ替える)。所持は最大12枚。
6. お供えは祭が進むほど増えていく。**どこまで福を呼べるか**=到達した祭の数と授かった総福銭がスコア。

### 御札とシナジー(一部)

| 札 | 効果 |
|---|---|
| 小判「両」 | そのまま4文。福の素。 |
| 招き猫「猫」 | 隣の小判1枚につき +5文。 |
| 達磨「達」 | 四隅に座ると 2倍。 |
| 鯛「鯛」 | 隣の鯛1枚につき +6文(並べるほど伸びる)。 |
| 富士「富」 | 同じ行と列の札すべて +1文。 |
| 米俵「米」 | 隣接する札1枚につき +3文(中央が得)。 |
| 鶴「鶴」/ 亀「亀」 | 鶴と亀が隣り合うと 3倍。 |
| 宝船「宝」 | 隣接する札の種類1つにつき +4文。 |
| 熊手「熊」 | 同じ行の他の札1枚につき +3文。 |
| 福袋「福」 | 回すたび2〜9文の気まぐれ。 |
| 桜「桜」 | 盤上の総額を 1.2倍(枚数ぶん乗算)。 |
| 鈴「鈴」 | 隣の鈴1枚につき +4文(鳴き交わす)。 |
| 門松「松」 | 角に座ると +5文。 |

位置(角・中央・行・列)と隣接が効くので、「寄せ」で並びを整える一手が点差になる。鯛を寄せ集める、小判で招き猫を囲う、鶴と亀を隣り合わせる——といったビルドを、授かる札で育てていくのがローグライトの面白さ。

## 操作

- **回す/収める:** 画面下の大ボタン(Space / Enter)。
- **寄せる:** 盤面を上下左右にスワイプ → その行/列がズレる。盤の四辺のガター矢印タップでも可。PCはキーボードの ←→(行)/ ↑↓(列)、`1`〜`3`で対象の行・列を指定。
- **音オン/オフ:** 右上の ♪ ボタン。

## 作成の背景

`/deep-research` による市場調査(2026年のヒット/バイラルした1人用ブラウザ・モバイルゲームのコアメカニクス)を起点に企画した。調査の要点:

- **勝ち筋は「誰もが知る既存の枠組み × たった1つの新鮮なひねり」。浅く入って深くハマる中毒ループが鍵。** Balatro はポーカー役という既知の核に無限のジョーカー/モディファイア合成を重ねて差別化した([Deconstructor of Fun](https://www.deconstructoroffun.com/blog/2025/10/06/how-do-you-find-and-scale-an-indie-hit-like-balatro))。
- **Luck Be a Landlord** は「デッキ構築をスロットマシンに置換」する1つのひねりで、シンボルの**隣接シナジー**(花+蜂、猫+ミルク等)からビルドが創発するローグライト・デッキビルダー。約91〜94%好評([Wikipedia](https://en.wikipedia.org/wiki/Luck_Be_a_Landlord) / [Fandom Wiki](https://luck-be-a-landlord.fandom.com/wiki/Luck_Be_a_Landlord))。
- 国内のミニゲーム圏でも「既知ジャンル × ローグライトのひねり」は強く、**将棋ライク(将棋×ローグライク)** は unityroom の人気ランキングでランク内最大の約214万PV([unityroom](https://unityroom.com/rankings/tags/116))。
- **スイカゲーム** は「同種を合体して進化」+ 落とす位置選びの2操作のみという極限のシンプルさ。縦画面タッチに最適で、ランキングが「もう一回」を駆動した([GAME Watch](https://game.watch.impress.co.jp/docs/kikaku/1537210.html))。
- **Wordle/Connections** は「答えを伏せた共有」「広告なしの極小UI」でバイラル化([Game Developer](https://www.gamedeveloper.com/marketing/josh-wardle-reflects-on-the-the-unconventional-road-to-wordle-s-success) / [NYT Connections (Wikipedia)](https://en.wikipedia.org/wiki/The_New_York_Times_Connections))。本作のリザルトも、授かった御札の漢字を並べたネタバレ控えめの共有テキストにした。

そこで「スロットのシンボル隣接シナジー(Luck Be a Landlord系)」という当サイト未開拓のジャンルを核に、**完全運だったスロットに「寄せ」=回した後に盤面を1マスずつズラして並びを整える一手介入**を足して差別化した。スロット × スライドパズルの小さな最適化が、毎スピンに能動的な意思決定を生む。

## 技術構成

- **単一HTMLファイル。** 外部ライブラリ・APIなし。札は絵文字でなく**漢字の木札(DOM)**で描画(縁起物の絵文字は環境差が大きいため。千社札コンセプトとも合致)。
- 盤・オーバーレイは DOM、効果音は **Web Audio でリアルタイム合成**(お囃子/拍子木/鈴を模した和風のトーン)。外部音源不使用。
- **iOS音声対策:** 初回タップで無音WAVを `<audio>` ループ再生してオーディオセッションを再生カテゴリへ、`AudioContext.resume()` の完了待ち、`onstatechange` で `interrupted` から自動復帰(`games/hakai/` の手法を踏襲)。
- ハイスコア(最高到達祭・総福銭)とミュート設定は localStorage(try/catch フォールバック)。
- **メインループはスタート画面の時点から成立する設計**で、ループが触る状態は全て宣言時に初期化(初回フレーム例外でループが死ぬのを防止)。
- スマホ対応:`viewport-fit=cover` + `env(safe-area-inset-*)`、`touch-action:none`(盤)、ダブルタップズーム抑止(`dblclick`/`gesturestart`/350ms以内の連続`touchend`を preventDefault)、入力は Pointer Events 統一(`click`非依存)。PCワイド画面対策に `max-width:min(460px,100vw)` の縦カラム+暗背景。
- OGP(1200×630)は、縦のプレイ画面を puppeteer で撮影し、朱漆背景+タイトル+「福」の朱印+スマホ枠に合成して生成。

## 実施したテスト

1. **構文チェック:** インラインスクリプトを抽出して `node --check`。
2. **バランス・シミュレーション:** スコア計算エンジン(`computeBoard`/`rentFor` 等)を実ファイルから抽出し、ボットで1万ラン。
   - 全情報の貪欲ボット(最適な寄せ+雑な買い物)→ 平均 約13祭。
   - EVベースのシナジー特化ボット(モンテカルロで購入を評価)→ **平均 約23.6祭・最高37祭**、26〜35祭にも2割が到達。
   - 「気軽に入って数分・本気のビルドで深くまで」という曲線と、明確なビルド/腕の差(平均13→23.6)を確認。お供えカーブはこの結果を見て、序盤は易しく中盤以降がなだらかに効く冪乗(`13.7·n^1.17`)へ調整した。
3. **実ブラウザE2E(puppeteer-core + ローカルChrome、ローカルHTTPサーバ経由):** モバイルビューポート(`hasTouch:true`)+タッチで自動プレイ。回す→寄せ→収める→お供え→御利益→次の祭、を40+スピン・13ショップぶん通し、`pageerror`/consoleエラーがゼロ、札のレンダリング(漢字)とスコア進行を確認。
   - 発見・修正したバグ:ショップの「札を納める/やめる」操作後に `#shopSub` を作り直すと `#shopCoins` 要素が消え、続く `renderShop` で null 参照になっていた → `renderShop` が `#shopSub` ごと再構築するよう修正。

## 開発で得た教訓

- **隣接シナジー系は「earnings がほぼ固定 vs ノルマが複利」で容易にハードウォールになる。** 初版はノルマが 1.34倍複利で全員が6〜7祭で全滅し、腕の差が出なかった。ノルマをなだらかな冪乗にし、所持上限を絞って(14→12)コンボの出現率を上げると、平均13・上手いと20〜37という健全なローグライト曲線になった。**「ボット1万ランで、腕の差(平均の開き)と到達分布の裾を見る」**のがこの手のゲームの調整の肝。
- 縁起物は絵文字の環境差が激しい(招き猫・達磨等は絵文字が無い)。**漢字の木札に振り切ると、クロスプラットフォームで崩れず、千社札という世界観にも合う。**

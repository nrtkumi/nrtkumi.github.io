# 駅名しりとり — 日本全国の本物の駅でつなぐ

前の駅の**最後の音**から始まる駅をつないでいく、しりとり。すべて**日本全国の実在する駅**で遊びます。入力すると本物の駅データから候補が出るので、知らない駅でも探せます。どれだけ長い「路線図」を伸ばせるか挑戦。

- **プレイ:** https://nrtkumi.github.io/games/ekishiri/
- **対応:** スマートフォン(タッチ・かな入力)推奨 / PC も可。縦画面・1人用・単一HTML。

---

## 遊び方

1. スタート駅から、最後の音(下の丸い「お題かな」)で始まる駅を探します。
2. 入力欄に**よみ(ひらがな)か漢字**を打つと、全国の駅から候補チップが出ます。タップでつなぐ。
3. つなぐたびに路線図が下に伸び、**1ターンの制限時間**(上のバー)はそのたびリセット。
4. **「ん」**で終わる駅を選ぶ or 時間切れ or ギブアップで終了。つないだ駅数がスコアです。

### ルールの細かいところ

- 同じ駅は二度使えません。
- **濁点・半濁点・小さい文字はゆるく判定**(が→か、ょ→よ など)。長音「ー」は無視して直前の音で判定。駅名の「〜駅」やカッコ書きは読みから除外。
- 「ん」で終わる候補は ⚠ つきの赤チップ。あえて選べばそこで打ち止め(プレイヤーの選択)。

### 評価

| つないだ駅 | 評価 |
|---|---|
| 25駅以上 | 🏆 鉄道神 |
| 15駅以上 | 🚄 特急なみの連結力 |
| 9駅以上 | 🚃 各駅停車で着実に |
| 5駅以上 | 🚉 まずは発車 |

自己ベスト(最長連結)は端末に保存されます。

## 作成の背景

**「Transit API で遊ぶ」**というお題に対し、日本語圏で誰もが知る言葉遊び**しりとり**を、辞書ではなく**実在の駅名データベース**で成立させました。

- **なぜ面白い/中毒性:** 「次の音」で記憶を探る快感と、候補チップで思わぬ駅に出会う**発見の楽しさ**。制限時間と「ん」の罠が near-miss を生み、最長記録の更新欲が「もう一本」を誘発します。
- **既存ゲームとの差別化:** サイトには Wikipedia のリンクを辿る「のりかえ百科」がありますが、本作は**駅名の音(かな)でつなぐ**言葉遊びで、API もメカニクスも別物。
- **データ源:** [Transit API](https://api.transit.ls8h.com) の `GET /api/v1/locations/suggest`。入力プレフィックスから駅名・よみ(`nameKana`)・座標を返すオートコンプリートを、そのまま候補生成とかな判定に利用。読み取り専用・認証不要・CORS開放。
- **片手・縦画面設計:** 入力欄と候補チップを最下部に置き、路線図は上方向に伸びて自動スクロール。

## 技術構成

- **単一HTML**(インラインCSS/JS、ビルド不要)。
- **デザインコンセプト:** 都市鉄道の**路線図**。クリーム色の紙地+ドットグリッド、縦に走る色違いのラインと駅ノード、丸ゴシック `Zen Maru Gothic` で温かみのある“路線図”モチーフ。
- かな正規化:カタカナ→ひらがな、小書き・濁点をマッチ用キーに畳み込み、`nameKana` 優先で読みを推定。読みが取れない駅は候補から除外。
- 効果音は Web Audio 合成、iOS 消音スイッチ対策(無音WAVループ + resume 待ち + interrupted 復帰)。スコアは localStorage。

## 実施したテスト

- インラインJSの構文チェック(`node --check`)。
- 実機相当の puppeteer E2E(iPhone ビューポート・タッチ、ローカルHTTPサーバ経由):自動プレイで実データの駅を6〜8駅連結し、入力→候補チップ→タップの実UI経路でもチェーンが伸びること、`pageerror`/console エラーが出ない(favicon の404を除く)ことを確認。
- 開発で得た教訓:`locations/suggest` の `nameKana` は**空の駅や、名前自体がかな/カナの駅、「〜駅」suffix を含む読み**が混在する。読み正規化と「読みが取れない駅の除外」が安定動作の鍵。

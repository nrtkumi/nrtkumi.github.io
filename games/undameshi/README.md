# うんだめし (UNDAMESHI)

裏向きのおみくじ札をめくって運勢ポイントを稼ぎ、**凶を引く前に「納める」か、もう1枚欲張るか**を選ぶおみくじチキンレース(push-your-luck)。3回の参拝ごとにお守りを授かって強くなるローグライト風の成長要素つき。

- **プレイURL**: https://nrtkumi.github.io/games/undameshi/
- **対応環境**: スマホ(縦画面・タッチ操作)ファースト。PCのマウス・キーボード操作にも対応。依存ライブラリなしの単一HTML。

## 遊び方

1. 「参拝する」をタップ。裏向きのおみくじ札が12枚並ぶ
2. 札をタップしてめくる。吉札なら運勢ポイントが「お供え」に貯まる
3. **「納める」をタップするとお供えが総運勢に確定**して次の参拝へ
4. 納める前に**凶**を引くとお供えは全部没収。**大凶**はさらにハートを1つ失う
5. ハートが尽きたらゲームオーバー。総運勢のベスト記録を更新していこう

### 駆け引きの要素

- 参拝開始時に「この箱: 凶3枚・大凶1枚入り」と構成が見える。めくった枚数から残りの凶率を読む
- 連続でめくるほどコンボ倍率が上がる(×1.0 → ×1.2 → ×1.4 …)。納めると倍率はリセット
- 参拝を重ねるごとに札の点数は上がるが、凶・大凶の枚数も増えていく
- 全部めくり切ると「全めくり」ボーナス

### 札の種類

| 札 | 基礎点 |
|---|---|
| 大吉 | +100 |
| 中吉 | +50 |
| 小吉 | +30 |
| 末吉 | +10 |
| 凶 | お供え没収 |
| 大凶 | お供え没収+ハート−1 |

(実際の獲得点は 参拝回数スケール × コンボ倍率 × 招き猫補正)

### お守り(3回の参拝ごとに1つ授かる)

| お守り | 効果 | 重ねがけ |
|---|---|---|
| 🐱 招き猫 | 吉札の運勢 +25% | ○ |
| 🏹 破魔矢 | 参拝ごとに1回、凶札・大凶札を無効化 | ○ |
| 🪞 御神鏡 | 参拝開始時に札が1枚透けて見える | ○ |
| 🔮 水晶玉 | 残りの凶札・大凶札の枚数が常に見える | × |
| 🧿 厄除け | 大凶でハートを失わなくなる | × |
| 🍶 御神酒 | いますぐハート+1(最大5) | ○ |
| 💰 賽銭箱 | 納めるとき、めくった札1枚につき+15ボーナス | ○ |
| 🦊 狐の面 | コンボ倍率の伸びが大きくなる | ○ |

## 作成の背景(市場調査の要点)

2026年6月時点の調査をもとに「占い」テーマで企画した。

- **占いアプリは2026年も定番ジャンル**: タロット・おみくじ・星座など多形式があり、「ゲーム感覚で占いを楽しむ」簡易系の需要が確認できる([マイベスト](https://my-best.com/6294)、[アプリブ](https://app-liv.jp/hobbies/uranai/3033/))→ 日本人に馴染み深い「おみくじ」をモチーフに採用
- **タロット・占い師ものはインディーゲームでも一ジャンル**: 『The Cosmic Wheel Sisterhood』(好評94%)や『Fortune Teller Simulator』など占い×ゲームメカニクスの組み合わせに前例多数([Steam](https://store.steampowered.com/app/1340480/The_Cosmic_Wheel_Sisterhood/)、[itch.io fortune-telling タグ](https://itch.io/games/tag-fortune-telling))→ ナラティブ系が多い中、本作はカジュアルな「リスク判断」に振った
- **ハイブリッドカジュアル化が2026年の主流**: 5秒で理解できる単一フックに進行・収集のメタレイヤーを重ねる構成が上位を占める([Game Growth Advisor](https://gamegrowthadvisor.com/blog/2026-04-16-hybrid-casual-game-design-strategy-2026/)、[Udonis](https://www.blog.udonis.co/mobile-marketing/mobile-games/casual-games))→ 「めくるか納めるか」の1フック+お守り強化のメタレイヤー

サイト内の既存ゲーム(ブロック崩しローグライト/マージパズル/タイミングタップ/神経衰弱/音楽クイズ/シューティング/2人対戦)とジャンルが被らない「push-your-luck(チキンレース)」を選んだ。

## 技術構成

- 依存ライブラリなしの単一HTML(canvas + Pointer Events、clickイベント不使用)
- フォントは既存ページと統一(Bungee + Zen Kaku Gothic New + Space Mono)
- 効果音はWeb Audioでリアルタイム合成(札めくり・吉のアルペジオ・凶のバズ・納めるコイン音など)
- **iOS消音スイッチ対策**: 初回タップ時に無音WAV(JSでArrayBuffer生成→Blob URL)を`<audio>`要素でループ再生してオーディオセッションを再生カテゴリへ切り替え、`AudioContext.resume()`のPromise完了を待ってアンロック判定、`onstatechange`で中断から自動復帰(games/hakai/と同方式)
- メインループ(rAF)はタイトル画面の時点から回し、ループが触る状態は全て宣言時に初期化
- dpr変換は毎フレーム `ctx.setTransform(dpr, 0, 0, dpr, 0, 0)` で再設定(コンテキスト喪失対策)
- ダブルタップズーム無効化の3点セット(`touch-action: manipulation` + dblclick/gesturestart抑止 + 連続touchend抑止)
- PCワイド画面ではルートを `min(100vw, 520px, 64vh)` の縦長カラムに制限して中央寄せ。キーボード操作(←→↑↓/WASD選択、Space/Enterめくり、Bで納める、お守りは1/2/3)対応
- ベスト運勢・ミュート設定はlocalStorage(try/catchでフォールバック)
- テスト用に読み取り+操作フック `window.__UND__` を公開(state/cardRect/flip/bank/pick)
- **キービジュアルはCodex CLI($imagegenスキル / gpt-image-2)で生成**: 「夜の神社・おみくじ箱から金色の札が舞い上がる・狐のお面・藍色#171028の夜空・アニメ調」のプロンプトで1024px PNGを生成し、`sips`で512px JPEG(約130KB)に変換。タイトル画面とゲームオーバー画面に表示し、読み込み失敗時は`onerror`で非表示にフォールバック
- OGP画像はゲームプレイのスクリーンショット(大吉の瞬間)+生成キービジュアルをHTML/CSSで合成し、puppeteerの`deviceScaleFactor: 2`+600×315クリップで1200×630を出力

## 実施したテスト

1. **構文チェック**: インラインscriptを抽出して `node --check` → OK
2. **ヘッドレスロジックテスト**: DOM/canvas/localStorageをスタブしてNode(vm)でゲームループを駆動。タイトル時点のループ生存→開始→札の構成検証→めくり→倍率→納める→ラウンド進行→凶バースト(没収・ハート維持)→お守り授与(3択生成・効果反映)→大凶連打でゲームオーバー→ベスト保存→リトライ初期化、まで全25項目PASS
3. **実ブラウザE2E**: puppeteer-core + ローカルChrome、モバイルビューポート(390×844、`hasTouch: true`)+タッチイベントで実プレイ。pageerror/consoleエラーゼロ、canvas描画ピクセルあり、タッチでめくり→納める→バースト→お守り選択→ゲームオーバー→localStorage永続化→リトライまで全17項目PASS。デスクトップ(1440×900)でも中央寄せカラム表示とキーボード操作(矢印+Space+B)を確認。スクリーンショットで見た目も目視確認

### 開発で得た教訓

- 「全部めくり切ったら自動で納める」のような遅延実行を仕込むとき、実行時点でもう一度phaseを確認しないと手動操作と二重発火する(bank関数のガードを `phase !== "play"` で統一して解消)
- ゲームオーバーまで自動プレイで誘導するテストでは、「最初に見つかった凶札」を踏むだけだとハートが減らない凶ばかり引いてタイムアウトする。大凶を優先して踏む・防御系お守りを避けて選ぶ、までテスト側で制御する必要があった
- 固定UI(ミュートボタン)を`position: fixed`で置くとPCワイド画面でゲームカラムから離れた画面隅に浮いてしまう。ゲームカラム(#stage)内の`position: absolute`にすると両環境で自然な位置になる

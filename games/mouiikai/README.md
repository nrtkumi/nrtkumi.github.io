# もういいかい — 夕暮れのかくれんぼ

夕暮れの空き地で、**鬼(それ)の視線を盗んで隠れ続ける**ステルス・かくれんぼ。茂みから茂みへ忍び、石を投げて鬼の気をそらし、光の輪=**ベース**にタッチすればラウンドクリア。クリアごとに強化を1つ選んで、どんどん難しくなる空き地を生き延びる **ローグライト** です。

- **プレイ:** https://nrtkumi.github.io/games/mouiikai/
- **対応:** スマートフォン(タッチ)推奨 / PC(マウス・キーボード)も可。縦画面・1人用・単一HTML。

---

## 遊び方

1. 画面を**ドラッグ**するとフローティングスティックが出て、その向きに移動します。**大きく弾くほど速い**ですが、走ると鬼に気づかれやすくなります。
2. **茂み(🌿)に入ると姿が消えます。** 茂みから茂みへ渡り歩いて鬼の視界をやり過ごしましょう。
3. **立ち止まると見つかりにくい**(だるまさんがころんだの要領)。鬼の視線が来たら止まって耐える/隠れるのが基本。
4. 右下の**🪨ボタン**で、向いている方向へ石を投げます。着地音で近くの鬼が**そちらを向いて寄っていく**ので、その隙に進めます(石は時間で補充)。
5. **上の発見メーター**が満タンになると「みいつけた!」=捕まってゲームオーバー。鬼の頭上の **? → !** が警戒度のサインです。
6. **ベース(光の輪)にタッチ**でラウンドクリア。早くクリアするほどスコアが高くなります。

### 鬼(それ)の3段階アラート

| サイン | 状態 | 挙動 |
|---|---|---|
| なし(黄色い視界) | 平静 | 巡回ルートを歩く |
| **?**(オレンジの視界) | 不審 | 立ち止まってこちらを向き、にじり寄る |
| **!**(赤い視界) | 発見 | メーター満タン → 捕まる |

不審のうちに視界から外れる/茂みに隠れると、鬼は警戒を解いて巡回に戻ります。

### 強化(クリアごとに3択から1つ)

| 強化 | 効果 |
|---|---|
| 👟 俊足 | 移動が速くなる |
| 🤫 忍び足 | 動いても見つかりにくい |
| 🎯 遠投 | 石が遠くまで届き、気をそらす範囲も広がる |
| 🪨 石拾い | 石の所持数+1・補充が速い |
| 😤 度胸 | 発見メーターの上昇が遅くなる |
| 🍃 息ひそめ | 立ち止まったときの隠密性が格段に上がる |
| 👁️ 心の目 | 鬼の視線が壁ごしでも見える |
| 🧧 お守り | 一度だけ捕まりを防ぐ(身代わり) |
| 👂 早耳 | 鬼がこちらを向く前に「…」の予兆が出る |

ラウンドが進むごとに、**鬼の数・速さ・視界の広さ**が増え、後半は**空き地が暗く**なって視界の光だけが頼りになります。

## デザインコンセプト

**「夕暮れの空き地・昭和のかくれんぼ」**。茜〜琥珀色の夕焼け、長く伸びる青い影、ザラついたレトロ絵本の質感。AI的になりがちな紫グラデ/ネオングローを避け、温かいサンセット配色 × ティールの影でまとめました。フォントは手書き風の **Yusei Magic**(見出し)+ 丸ゴシックの **Zen Maru Gothic**(本文)。タイトルは、かくれんぼの掛け声「もう いいかい?」「まあだ だよ」から。

## 作成の背景(市場調査)

`/deep-research` による調査で、2026年のモバイル/ブラウザ向け1人用ゲームのトレンドを確認し、本作の設計に反映しました。要点と出典:

- **視線回避ステルス × ローグライト(片手操作)** — *Stealth Master*(SayGames)が代表例。**各ガードの視野を地面に可視化**し、赤ゾーンに入ると即発見、**指スライドの片手操作**、フロア間で**3択の一時強化**を選び、進むほど敵が増えて難化する、というローグライト構造。本作の「視野コーン+ベース到達+クリア毎の強化選択」は、この型をかくれんぼに翻案したもの。([Stealth Master / Uptodown](https://stealth-master.en.uptodown.com/android))
- **公平なステルス検知の定石** — 突然の発見は理不尽に感じられるため、**平静 / 不審 / 警戒の3段階アラート**にし、**敵のアニメ+頭上インジケータ**など複数チャネルで発見を予告(テレグラフ)する設計が定石。本作の **? → !** マークと視界の色変化、メーター表示はこれに倣っています。([Game Design Skills: Stealth](https://gamedesignskills.com/game-design/stealth/))
- **かくれんぼ/鬼ごっこのデジタル化と「止まる」緊張感** — Squid Game由来の *Red Light Green Light*(動く/止まるのタイミング)がTikTokで桁外れにバイラル化(#SquidGame は110億回再生超)。本作の「**立ち止まると見つかりにくい**」は、この move/freeze の緊張感を取り込んだもの。([Capital FM](https://www.capitalfm.com/news/tv-film/squid-game-tik-tok-red-light-green-light/) / [elgoog: Squid Game](https://elgoog.im/squid-game/))
- **短時間カジュアルのローグライト構造** — *Vampire Survivors* 型の「レベルアップごとに3〜4択の強化」「操作は移動のみ」「終盤の強制エスカレーション」は、短いセッションに中毒性を与えるテンプレ。本作のクリア毎3択強化・移動主体の操作・後半の難化はこれを参照。([Vampire Survivors / Wikipedia](https://en.wikipedia.org/wiki/Vampire_Survivors))
- **ワンフィンガー操作の実装核** — タッチ座標(`e.touches[0]`)を直接プレイヤー操作にマップし、`preventDefault()` / `touch-action:none` でブラウザのスクロール・ドラッグを抑止するのが基本。([MDN: Mobile touch controls](https://developer.mozilla.org/en-US/docs/Games/Techniques/Control_mechanisms/Mobile_touch))

> 着想の参考: *Meccha Chameleon*(2026・カムフラージュ型かくれんぼ、5日で200万本)や Squid Game S3 の "Hide and Seek"(赤=鬼/青=隠れ手の非対称・鍵集め脱出)も、隠れる遊びの新しい形として調査に挙がりました。

## 技術構成

- **単一HTMLファイル**。描画は素の Canvas 2D(トップダウン、ロットを画面にフィット)。外部ゲームライブラリなし。
- **視界判定:** 鬼ごとに扇形の視野コーン(角度・半径)。線分 vs AABB の交差で**壁による遮蔽(LOS)**を判定。茂みは LOS を遮らず、入ると潜伏。
- **発見ロジック:** 鬼ごとに警戒度を蓄積。距離(近いほど速い)× 動作(走る>歩く>静止)で上昇、視界を外れると減衰。最大値を発見メーターに表示。
- **マップ生成:** シード付き乱数で塀・物置・茂み・鬼の巡回ルートを毎ラウンド生成。**グリッドBFSで「開始→ベース」の到達可能性を検証**し、解けない配置は引き直し。
- **操作:** Pointer Events 統一のフローティングスティック+石投げボタン。`click` 非依存、ダブルタップズーム抑止(3点セット)、セーフエリア対応、PC用キーボード操作も実装。
- **音:** Web Audio でリアルタイム合成(足音・葉ずれ・石・発見音・夕方のチャイム風クリア音)。**iOS消音スイッチ対策**(無音WAVを `<audio>` でループ、`AudioContext.resume()` の完了待ち、`interrupted` からの自動復帰)込み。
- **永続化:** 最高記録(ラウンド/スコア)を localStorage に保存。
- **画像アセット:** タイトル/OGPのキービジュアルに加え、**ゲーム内のスプライトも Codex 経由の gpt-image で生成**(`assets/spr/`):
  - 地面テクスチャ(夕暮れの土)・板塀の木材テクスチャ … タイル状に敷き詰め
  - 隠れ手の子供・鬼(赤いお面+両腕を広げて探す)・茂み・ベースの注連縄付き祠 … いずれも**真上から見たトップダウン視点**で生成
  - gpt-image は透過非対応のため、**マゼンタ(#FF00FF)の単色背景で生成 → ヘッドレスCanvasでクロマキー+デスピル+バウンディングボックスで切り抜き**して透過PNG化(`title`画風を固定の前置きにして6枚を統一感のあるバッチ生成)
  - キャラは `face` 方向に回転させて描画。**全スプライトは読み込み失敗時にベクター描画へフォールバック**するので無言で壊れない
  - 注意: 並列生成だと Codex が保存ファイル名を取り違えることがある(木材テクスチャに別画像が入った)。**1枚ずつ生成し直して確認**するのが安全
- **OGP** は実プレイ画面(スプライト適用後)をスマホ枠に合成した 1200×630 カード。

## 実施したテスト

1. **構文チェック:** インラインスクリプトを抽出して `node --check`。
2. **実ブラウザ E2E(puppeteer-core + ローカルChrome、モバイルビューポート+タッチ、HTTPサーバ経由):**
   - `pageerror` / console エラーが**ゼロ**(favicon の404を除く)。
   - canvas に描画ピクセルがあることを `getImageData` で確認(非空・多色)。
   - フロー全通し: タイトル→開始→**ベース到達でクリア→強化選択→次ラウンド(スコア加算)**→**視界に晒され続けて捕まる→ゲームオーバー**→リトライ。
   - **茂みから出られること**を座標で検証(過去の自バグ「茂み内で速度0=出られない」の回帰防止)。
3. **目視:** タイトル/プレイ画面のスクリーンショットで配色・視界コーン・UIを確認。

### 開発で得た教訓

- **隠れ場所は「入ったら出られる」が大前提。** 当初「茂み内は移動不可(=完全潜伏)」にしたら茂みから一生出られない詰みになった。潜伏は「常に隠れるが移動はできる」、緊張は**茂みとベースの間のギャップ**で作るのが正解だった。
- **ステルスは視界の可読性が命。** 視野コーンの初期実装は半透明すぎてほぼ見えず、ゲームとして成立しなかった。塗りを濃く+縁取りを足して常に形が読めるようにした。
- ループに触れる状態は宣言時に全初期化し、メインループはタイトルから常時回す(初回フレーム例外でループが死ぬ事故の回避)。`touchend` の `preventDefault` は **`cancelable` を確認**してから呼ぶ(非キャンセル時の警告を回避)。

# BARASHI（バラシ） — ネジ外しローグライト

**むき出しのネジだけタップで外せる。** 外したネジは下の作業トレーへ送られ、同じ色が **3本そろうとバラし成功＝スコア爆発**。空きトレーが無い色は右の「あふれ棚」へ逃がすしかなく、**あふれ棚が満杯のまま置けなくなったら一巻の終わり**。盤面を全部バラすとクリアし、稼いだ **BOLT** で工具を買い足して次の階へ——色も段数も増え続けるエンドレスのネジ外しパズル。

- **プレイURL**: https://nrtkumi.github.io/games/barashi/
- **対応環境**: スマホ（縦画面・タッチ）推奨。PC（マウス＋キーボード）も可。単一HTMLでオフライン動作。

---

## 遊び方

1. **上にパネルが乗っていないネジ（＝明るく色づいたネジ）だけ** タップで外せる。グレーのネジは上のパネルに覆われていて外せない。
2. 外したネジは下の **作業トレー** へ自動で運ばれる。同じ色が **3本** そろうと **バラし成功**（スコア＆BOLT獲得、コンボ加算）。
3. どの作業トレーにも置けない色は、右下の **あふれ棚** に逃がす。あとで同色トレーが空くと自動で再仕分けされる。
4. **あふれ棚が満杯のまま、どこにも置けないネジを外すと GAME OVER。** ＝「あふれた」。
5. パネルを全部バラすと **LV クリア**。WORKBENCH（ショップ）で工具を買い、次の階へ降りる。

### コアループ（5秒で理解できる操作）
> むき出しのネジを外す → 同色3本でバラす → あふれる前に盤面を空にする。

「外す一手」はすべてプレイヤーの選択。あふれ棚があと1枠……という **“惜しさ” は運ではなく自分の読み違い** として立ち上がる設計にしている（後述の調査根拠）。

### 工具（ショップでBOLTと交換）

| 工具 | 効果 | 種別 |
|---|---|---|
| 🗄️ あふれ棚 +1 | あふれを置ける棚を1枠増やす。生存の保険 | 恒久（最大9） |
| 🧰 作業トレー +1 | 同時に仕分けられる色を1つ増やす。詰みにくくなる | 恒久（最大6） |
| 🧲 マグネット ×2 | むき出しの同色ネジを一気に外す | 消費（2回） |
| ↩️ 巻き戻し ×2 | 直前の1手を取り消す | 消費（2回） |
| 🧹 棚そうじ | あふれ棚を空にして再仕分け | 消費 |

ショップは毎回「あふれ棚＋作業トレー」＋ランダムな消費工具2種を提示。階が深いほど恒久強化は割高になる（pick rate / 勝率相関で調整するデッキビルダーのバランス論に倣い、保険系を取り続けるだけでは伸び悩むようコスト逓増）。

PCでは **M＝マグネット / U＝巻き戻し**。

## 作成の背景（市場調査）

`/deep-research`（複数ソースのファンアウト検索＋3票の敵対的検証）で 2025–2026 のブラウザ/モバイル1人用ゲームのトレンドを調査し、当サイトの既存20作と被らない切り口を選んだ。要点：

- **空間ソート/占有解消パズルがカジュアル市場を支配**。Block Blast! は 2025年DL首位（約3.68億DL、前年比+65.7%）、Color Block Jam・Screwdom が収益上位。とくに **ネジ抜きパズル**は「上にあるものしか取れない（レイヤー占有解消）＋限られたドックに一時保管＝満杯で失敗」というコアループで、2026年に “screw puzzle gold rush” と呼ばれるほど伸びた（[Deconstructor of Fun](https://www.deconstructoroffun.com/blog/2026/2/20/the-screw-puzzle-gold-rush) / [Gamigion](https://www.gamigion.com/color-block-jam-is-the-most-successful-new-puzzle-in-2025/) / [MAF: Top Mobile Games 2025](https://maf.ad/en/blog/top-mobile-games-2025/)）。当サイトの既存20作にこのジャンルは皆無だったため、**ネジ外し＝バラシ** を中核に採用。
- **中毒ループの核は「指数的に上がる目標＋ラウンド間ショップでの強化」**（Balatro）。既知メカニクスを入口にし、敗北を「不公平」でなく「自分の誤算」として枠付けると再挑戦欲が湧く（[ejaw: Balatro](https://ejaw.net/balatro/) / [Error&Exp](https://errorandexp.substack.com/p/unpacking-balatros-addicting-game)）。→ 階ごとに色数・段数・ネジ総数を増やし、間にWORKBENCHを挟むローグライト構成に。
- **ニアミス（惜しい外れ）は満足度を下げつつ“もう一回”欲を上げる。ただしプレイヤー自身が操作を選択している＝主体性がある場合に限る**（[Clark et al. 2009, Neuron](https://www.ncbi.nlm.nih.gov/pmc/articles/PMC2658737/)）。→ あふれ棚の残り枠という「自分の手で招く惜しさ」を緊張の中心に据えた。ガチャ的な運頼みの惜しさは入れていない。
- デッキビルダーのバランスは **カード選択率＋勝率相関** の2指標で詰める（[Game Developer: Slay the Spire](https://www.gamedeveloper.com/design/how-i-slay-the-spire-i-s-devs-use-data-to-balance-their-roguelike-deck-builder)）。→ 恒久強化のコスト逓増で「保険を取り続けるだけ」を弱め、消費工具との取捨選択を促す。

（注：調査では「可変比率報酬がループを強める（Skinner）」「ネジ抜きの中毒因＝隠れ情報」などの説は敵対的検証で却下されたため、設計の根拠には採用していない。）

## デザインコンセプト

**整備工のワークベンチ（鈍く光るスチール＆木目の作業台）。** AI生成にありがちな紫グラデ/グラスモーフィズム/ネオングローを避け、無彩色のブラッシュド・メタルと木目パネル、ハザードイエロー（#f5c518）のアクセントで“工具箱”の質感に寄せた。

- タイポ：見出しは工業ステンシルの **Saira Stencil One**、数値は **Saira**、日本語UIは **Zen Kaku Gothic New**。
- ネジは中心十字スロット＋放射グラデで金属球感。**外せるネジは原色＋ハイライト／覆われたネジはグレー** で可否を一目で判別。
- バラし成功で確定するハザードイエローのフラッシュとパーティクル、コンボでピッチが上がる効果音。

## 技術構成

- **単一HTMLファイル**（外部アセットはGoogle FontsとOGP画像のみ）。canvas 2D で描画、UIオーバーレイはDOM。
- **盤面生成**：各色のネジ本数を必ず3の倍数にして「理論上クリア可能」を保証。パネルは軸平行の角丸矩形をランダムに重ね、上位zのパネルがネジ頭を覆っているかで「外せる/覆われている」を毎フレーム判定（点 in 矩形）。
- **iOS消音スイッチ対策**：初回タップで無音WAVを `<audio>` ループ再生しオーディオセッションを再生カテゴリへ、`AudioContext.resume()` 完了待ち、`onstatechange` で `interrupted` から自動復帰（`games/hakai` の手法を踏襲）。効果音はWeb Audioでリアルタイム合成。
- **スマホ対応**：`viewport-fit=cover`＋`env(safe-area-inset-*)`、`touch-action`、ダブルタップズーム無効（350ms以内の連続`touchend`をpreventDefault、`dblclick`/`gesturestart`も抑止）、入力は **Pointer Events**（`click`非依存）。
- **PCワイド対策**：ルートを `max-width:min(480px,62vh)` の縦長カラムに制限・中央寄せ。キーボード操作（M/U）も用意。
- **メインループは起動時から回り**、ループが触る状態はすべて宣言時に初期化（初回フレーム例外でループが死ぬ事故の防止）。`draw` は毎フレーム `ctx.setTransform(DPR,…)` でDPR変換を再設定（iOSのコンテキスト喪失・スクショ撮影対策）。
- **永続化**：ハイスコア（SCORE / 到達LV）とミュート設定を localStorage（try/catchフォールバック）。
- OGP一式（絶対URL・1200×630）を実装段階で埋め込み済み。OGP画像は実プレイ画面を撮影してキービジュアルに合成。

## 実施したテスト

1. **構文チェック**：インラインscriptを抽出して `node --check`。
2. **ヘッドレスロジックテスト**（DOM/canvas/AudioをスタブしてNodeで実行）：
   - 12レベル分の盤面生成不変条件（各色のネジ本数が3の倍数＝クリア可能、空パネルなし、開始時に必ず外せるネジが存在）。
   - 貪欲ソルバが十分な枠で盤面を完全クリアできる。
   - 枠を絞ると「あふれ」で GAME OVER に至る。
   - あふれ→GAME OVER でハイスコアが localStorage に保存される。
3. **実ブラウザE2E**（puppeteer-core ＋ ローカルChrome、`python3 -m http.server` 経由、モバイル`hasTouch`ビューポート）：
   - スタート→実タッチでネジを連打→スコア/コンボ/BOLT加算を確認。
   - canvasに描画ピクセルあり（約110万px）、`pageerror`/consoleエラーは favicon の 404 以外ゼロ。
   - 盤面クリア→WORKBENCH（4枚提示）→購入→次階（LV2）への遷移を実タッチ/Pointerで確認。
   - タイトル・LV1/LV6盤面・ショップのスクリーンショットを目視確認。

## 開発で得た教訓

- **`file://` ではなくローカルHTTPサーバ経由でE2E**：同梱画像でcanvasが汚染されると `getImageData` が `SecurityError` になるため。
- **覆い判定は「点 in 矩形」で十分軽い**。パネルを軸平行矩形に限定したことで、回転表現は描画時のみに留め、当たり判定の正確さを優先できた。
- **生成時に各色を3の倍数で割り当てる**だけで「理論上クリア可能」を保証でき、難易度は枠（トレー/あふれ棚）の絞り込みで自然に作れる。
- ヘッドレステストでは `levelClear` の `setTimeout` がNodeで発火せず内部ロック（`lockInput`）が解けないため、テスト用の解除フックを用意して各テスト間でリセットした。

# シキワケ（SHIKIWAKE） — サムネ色分けスワイプ

**次々流れてくる動画サムネを、支配的な色の方へ素早く仕分ける。** ←寒色／↑モノクロ／→暖色 にスワイプ（またはボタン）で判定。連続正解でコンボ倍率が上がり、速くなるほど忙しくなる反射神経スコアアタック。正解は**サムネ画像から計算した支配色**なので必ず1つに定まる。

- **プレイURL**: https://nrtkumi.github.io/games/shikiwake/
- **対応環境**: スマホ（縦画面・スワイプ／タップ）ファースト。PC（←↑→キー、マウス）対応。単一HTML。
- **通信**: 必要（YouTubeのサムネ画像とIFrame Playerをブラウザから直接利用）
- **デザインコンセプト**: 「色の分別レーン」。暗い舞台に、暖色＝オレンジ／寒色＝ブルー／モノクロ＝グレーの三色を効かせ、見出しは丸ゴシックの RocknRoll One。流れるサムネがそのまま絵になる。

## 遊び方

1. 中央のサムネカードを見て、**支配的な色**を判断する。
2. **スワイプ**（←寒色／↑モノクロ／→暖色）か、下の**3ボタン**で仕分ける。
3. 正解すると加点＆**コンボ**。4連続ごとに倍率が上がる（×1.5, ×2.0 …）。速く答えるほどボーナス。
4. 間違い・時間切れで**ミス**（❤️が減る）。コンボはリセット。3ミスで終了。
5. 仕分けるほどカードの**制限時間が短く**なる。ベストスコアは保存。ゲームオーバー後はさっきの動画を本編再生できる。

### 色の判定基準（計算ルール）

- **モノクロ**: 彩度が低い（白黒・くすみ）サムネ。
- **暖色 / 寒色**: 彩度がある場合、平均色相が赤〜黄なら**暖色**、緑〜青〜紫なら**寒色**。

## 作成の背景

当初は「音楽／ゲーム／料理…のジャンル仕分け」を想定したが、**ジャンルが見た目で判別でき、かつ埋め込み可能なサムネ**を十分な数そろえるのが難しかった（人物の顔サムネが多く視覚的に区別しづらい／非音楽の有名動画はID・埋め込み可否が不安定）。そこで「**必ず正解が1つに定まり、見た瞬間に判断できる**」軸として、画像から計算する**支配色**に切り替えた。これなら判定がブレず、サムネ画像そのものを手がかりにする反射神経ゲームとして成立する。色分けの三択は、配色しきい値をボット計測でほぼ均等（暖色:モノクロ:寒色 ≒ 14:7:7）になるよう調整した。

## 技術構成

- 単一HTML＋CDN（Google Fonts: RocknRoll One / Noto Sans JP、YouTube IFrame API）。カードはDOM、タイマーは`requestAnimationFrame`。
- 色分類: 起動時に全サムネ（`i.ytimg.com/.../hqdefault.jpg` を `crossOrigin="anonymous"`）の中央16:9領域を24×14に縮小して `getImageData`、平均RGBから彩度・色相を算出して暖色/モノクロ/寒色に分類。`i.ytimg.com` は `access-control-allow-origin:*` なので汚染されない。
- 入力はPointer Eventsのドラッグで3方向判定（←↑→）。ボタン・キーボードでも操作可。`click`非依存。
- ゲームオーバー後の本編再生は IFrame Player API（`loadVideoById`）。プレーヤー前面にオーバーレイしない。収録動画は oEmbed で実在＆埋め込み可を確認済み。[[youtube-embed-gotchas]] の構成を踏襲。
- iOS対策一式（無音WAVアンロック、`resume()`待ち、`touch-action`、ダブルタップズーム無効）。効果音はWeb Audio合成、ベストは localStorage。

## テスト

- **構文チェック**: `node --check`。
- **配色分布の計測**: 全サムネの平均色をヘッドレスで取得し、しきい値を 0.13〜0.17 でスイープ。0.15 が三択ほぼ均等（14:7:7）と判明して採用。
- **実ブラウザE2E**（puppeteer-core、モバイルviewport＋タッチ）: スタート→（テストフックで）連続正解→コンボ倍率上昇、ミスでライフ減・終了、`pageerror`ゼロ、`getImageData`がサムネ由来（汚染なし）で分類できること、終了後に本編プレーヤーがロードされることを確認。スクリーンショットで見た目も目視確認。

### 教訓

- 「見た目で当てさせる仕分け」は、**カテゴリが視覚的に判別可能**で、かつ素材が十分そろうことが前提。素材（埋め込み可・カテゴリ明快なサムネ）が確保できないときは、**画像から計算できる客観軸（支配色）**に振ると、判定がブレず公平で実装も安定する。

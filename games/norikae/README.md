# のりかえ百科 — ことばの路線図でゴールへ

**Wikipediaのリンクだけ**を乗り継いで、スタートの言葉からゴールの言葉へたどり着く「WikiRace（ウィキ競争）」パズル。記事から記事へ“乗り換え”を繰り返し、**できるだけ少ない手数**でゴールを目指します。すべて **日本語版Wikipedia の生きた記事**で遊びます。

- **プレイ:** https://nrtkumi.github.io/games/norikae/
- **対応:** スマートフォン(タッチ)推奨 / PC(マウス)も可。縦画面・1人用・単一HTML。

---

## 遊び方

1. 出されたお題(例: **寿司 → 宇宙**)で「出発」。
2. 今いる記事の中にある**リンク(行き先)をタップ**すると、その言葉へ乗り換え。
3. これを繰り返して**ゴールの言葉**にたどり着けばクリア!
4. 行き先が多いときは**検索ボックスで絞り込み**。「もどる」で1手戻る、「新しいお題」で別の路線へ。

### 評価

| のりかえ回数 | 評価 |
|---|---|
| 2手以下 | 🏆 達人級・最短ルート |
| 3〜4手 | ✨ お見事 |
| 5手以上 | 🚶 ゴール |

自己ベスト(最少のりかえ)は端末に記録されます。**コツ:** ゴールに関係しそうな“大きな言葉”(国・分野・有名な物事)を経由すると近道になりやすい。

## 作成の背景

「Wikipediaの公開APIで遊ぶ」というお題に対し、**WikiRace**(別名 Wikipedia race / wikiracing)を採用しました。リンクだけを辿ってゴール記事を目指す、Wikipedia文化として古くから親しまれてきた遊びです([Wikipedia:Wikirace](https://en.wikipedia.org/wiki/Wikiracing))。

- **なぜ面白い/中毒性:** 「あと1回の乗り換えで着くかも」という near-miss と、最少手の自己ベスト更新欲が「もう1お題」を誘発します(ハイスコア更新ループ)。記事を読み解いて近道を探す**発見の快感**も核。
- **既存ゲームとの差別化:** サイトには Wikimedia の*閲覧数*を当てる pedia-battle がありますが、本作は**記事のリンク構造をたどるナビゲーション**で、API も遊びも別物。WikiRace 形式はサイト初。
- **データ源:** [MediaWiki Action API](https://ja.wikipedia.org/w/api.php)(`prop=links`)で行き先リンクを、[REST API](https://ja.wikipedia.org/api/rest_v1/)(`page/summary`)で記事の要約・サムネ・正式名称(リダイレクト解決)を取得。いずれもキー不要・CORS対応(`origin=*`)。
- **片手・縦画面設計:** リンクはチップ状で親指の届く下半分に([thumb zone](https://www.smashingmagazine.com/2020/02/design-mobile-apps-one-hand-usage/))。

## 技術構成

- **単一HTML**(インラインCSS/JS、ビルド不要)。
- **取得:** 記事ごとに `prop=links`(`plnamespace=0`, `pllimit=max`, 最大2ページ継続=最大約900リンク)+ `page/summary` を並列取得し、訪問済み記事は **Mapにキャッシュ**(「もどる」や再訪が即時)。
- **ノイズ除去:** 「○○年」「○世紀」「○千年紀」「○年代」「○○年の〜」「曖昧さ回避」等の日付・暦リンクを除外し、意味のある行き先だけを表示(「1964年東京オリンピック」のような固有イベントは残す)。
- **正誤/リダイレクト:** ゴール判定は `summary` の正式タイトル(redirect解決)で比較。行き止まり(存在しない記事)はトーストで通知して進まない。
- **デザインコンセプト:** 「ことばの路線図(トランジットマップ)」— 生成りの紙地にドットグリッド、路線色(赤青緑橙紫)のチップ、現在地→ゴールのルートバー。明朝の角丸見出し+等幅の計器類。AI的な紫グラデ/グラスモーフィズムは不使用。
- **iOS対応:** Web Audio の消音対策(無音WAVループ+`resume()`待ち+`onstatechange`復帰)、効果音は合成。ダブルタップズーム無効化(ただし検索欄のフォーカスは妨げない)、Pointer Events 統一(`click`非依存)、セーフエリア対応、PCワイド対策(縦カラム中央寄せ)。検索リストは `touch-action:pan-y` で縦スクロール可。
- **永続化:** localStorage に自己ベスト・クリア数(try/catch)。

## 実施したテストと教訓

1. **構文チェック:** インラインJSを抽出し `node --check` → OK。
2. **ヘッドレスロジックテスト:** お題プールの距離、`isJunk`(日付/世紀/千年紀を除外し固有イベントは保持)、ペアの相異、時間整形、評価ティアを検証 → 6/6 PASS。
3. **実ブラウザE2E(puppeteer-core + Chrome, モバイルVP `hasTouch`, ライブWikipedia):** 出発→**リンクチップ描画**→検索絞り込み→**乗り換えで記事遷移(hops増加)**→もどる→**ゴール到達でクリア**(お題を引き直してゴールが直接リンクされた路線で勝利)まで確認。`pageerror`/コンソールエラーは favicon 以外ゼロ。スクリーンショット目視OK。

**教訓:** Wikipediaの `prop=links` は**アルファベット順**で返るため、何もしないと「年・世紀・日付」が先頭に大量に並ぶ。**日付系を除外**しないと行き先リストが実用にならない。記事はキャッシュして「もどる」を即時化。`origin=*` を付ければ Action API も CORS で叩ける。

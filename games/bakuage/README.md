# 爆アゲ（BAKUAGE） — 登録者100万人への一発逆転

**バズった神動画を「元ネタ」に動画を投稿し、ビューを稼ぐ。** もう一本撮るほど稼げるが、続けるほど**炎上→爆死**の確率が上がる。稼ぎを「寝て確定」すれば登録者になる——**30日間**で底辺YouTuberから**伝説（登録者1億）**を目指す、プッシュ・ユア・ラックの経営ローグライト。元ネタ札には実在の有名MVサムネを使い、到達したランクの神動画は**本編を再生**できる。

- **プレイURL**: https://nrtkumi.github.io/games/bakuage/
- **対応環境**: スマホ（縦画面・タッチ）ファースト。PCのマウスにも対応。単一HTML。
- **通信**: 必要（YouTubeのサムネ画像とIFrame Playerをブラウザから直接利用）
- **デザインコンセプト**: 「クリックバイト・サムネ」。クリーム色の紙地に極太の黒アウトライン、赤・黄・緑のステッカー配色、見出しは Dela Gothic One。バズ動画のサムネ感をそのままUIに。

## 遊び方

1. 毎日、3枚の**元ネタ札**（実在MVのサムネ）が配られる。⚡稼ぎ・✨バズ期待・🔥炎上リスクのバランスを見て1枚を選び**投稿**。
2. 投稿すると「今日の再生数」が伸びる。たまに**バズ**で再生数が一気に跳ねる。
3. 投稿後に2択：
   - **🎬 もう一本** — 同じ日にもう1本投稿。再生数は乗算で伸びるが、**2本目以降は炎上で爆死する確率**（表示される％）が毎回上がる。
   - **😴 寝て確定** — その日の再生数を**登録者**に変換して銀行入金。安全。
4. 炎上で爆死すると今日の稼ぎは消滅し、**モチベ（❤️×3）**を1つ失う。モチベがゼロになると**引退（ゲームオーバー）**。
5. 登録者がしきい値を超えると**ランクアップ**（モチベ+1・神動画解放）。30日終了かモチベ消滅で結果発表。ベスト登録者数は保存。

### ランク（神動画報酬つき）

底辺 → 駆け出し（1万）→ 中堅（10万）→ 人気（50万）→ トップ（200万）→ カリスマ（1000万）→ **伝説（1億）**

## 作成の背景

YouTube題材として「再生数を当てる／流す」以外の切り口を探し、**“バズの不確実性”そのものをゲームにする**方向にした。元ネタ札に実在のバズMVサムネを使うことで「バズった動画を参考に自分もバズる」という配信者の営みを擬似体験できる。コアは古典的なプッシュ・ユア・ラック（Pig/チキンレース系）で、操作は「もう一本 or 寝る」の2択に凝縮。スマホで5秒で理解できる。

### バランス調整（重要な学び）

設計当初は**加算モデル**（1投稿で固定量を加算）だったが、ボット1万ラン以上のシミュレーションで「**安全に2本で止めるのが常に最適**＝押し引きが無意味」と判明。

- そこで**乗算モデル**（投稿ごとに今日の再生数を×1.5〜2.7倍に伸ばし、全額が爆死リスクに晒される）へ変更。これでプレス・ユア・ラック本来の「いつ降りるか」の内点最適が生まれた。
- ただし乗算だと「1本だけ確定×無限日」が無リスクで勝ててしまうため、**30日のデイリミット**を導入。安全プレイは時間内に伝説へ届かず、押せば早いがモチベと貴重な日数を失う、という緊張に着地。
- 最終バランス（ボット検証）: 安全寄りは中堅〜人気で着地、攻めて深さ3まで押すと伝説到達が約4.5%（高変動）。深く押すほど序盤で爆死しやすい、狙いどおりの分布。

## 技術構成

- 単一HTML＋CDN（Google Fonts: Dela Gothic One / Noto Sans JP、YouTube IFrame API）。
- 元ネタ札・報酬はすべて oEmbed で実在＆埋め込み可を確認済みの動画。サムネは `i.ytimg.com/.../hqdefault.jpg`。
- 報酬再生は IFrame Player API（`loadVideoById`）。プレーヤー前面にオーバーレイしない。[[youtube-embed-gotchas]]の構成を踏襲。
- 確率・乗算ロジックはチューニング定数を切り出し、ヘッドレスのボットシミュレーションで反復調整。
- iOS対策一式（無音WAVアンロック、`resume()`待ち、ダブルタップズーム無効、`touch-action:manipulation`）。効果音はWeb Audio合成、ベストは localStorage。

## テスト

- **構文チェック**: `node --check`。
- **ボットシミュレーション**: 加算/乗算モデル × 各プッシュ深さ × 各リスクキャップで数万ラン回し、到達ランク分布・勝率・日数・押し引きの最適点を可視化して定数決定。
- **実ブラウザE2E**（puppeteer-core、モバイルviewport＋タッチ）: スタート→カード選択→もう一本/寝るを自動で繰り返し、`pageerror`ゼロ、登録者が増える、爆死→引退まで一連が破綻しないことを確認。殿堂から本編プレーヤーがロードされることも確認。

### 教訓

- プッシュ・ユア・ラックは**加算だと最適が常に「即降り」**になり押し引きが死ぬ。**乗算（全額が次の賭けに乗る）＋資源/時間の上限**が緊張を生む鍵。ボット検証なしでは気づけなかった。

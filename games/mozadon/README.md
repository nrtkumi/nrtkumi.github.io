# モザドン — MOZADON

モザイクの奥に隠れた**本物のミュージックビデオ**を、くっきり映る前に**4択で早押し**して当てる音楽クイズ・ローグライト。正解するとそのMVが**本物で流れる**。連続正解で倍率が伸び、㊙(ライフ)が尽きるまでエンドレス。世界のヒット曲＆J-POP 約60曲からランダム出題。

- **プレイURL**: https://nrtkumi.github.io/games/mozadon/
- **対応環境**: スマホ(iOS Safari / Android Chrome)・PCブラウザ。タッチ操作のみで完結。縦画面。
- **デザインコンセプト**: 深夜の音楽番組／VHSの録画画面 — 暗い舞台にネオン(マゼンタ×シアン)、走査線とノイズ、`REC`ランプ、`Orbitron`の7セグ風スコア。生成AIっぽい紫グラデ＋ガラス質感は避け、音楽TVの質感に寄せた。

## 遊び方

1. **スタート**を押すと、画面に大きなモザイク(モザイク化したYouTubeサムネ)が現れ、時間とともに**少しずつくっきり**していく。
2. 下の**4つの選択肢**から、その曲だと思うものをタップ(またはキー `1`〜`4`)。
3. **早く・荒いうちに当てるほど高得点**(1000点 → だんだん減って最低100点)。
4. 正解すると**本物のMVがその場で再生**され、不正解だと正解が表示される。`▶ つぎへ`(またはSpace)で次へ。
5. 間違えると㊙が1つ減り、**㊙が尽きるとゲームオーバー**。連続正解で**コンボ倍率**が伸びる。
6. **5問正解ごとにアイテム**を1つ選んで強化(ローグライト)。レベルが上がると、モザイクが解けるのが速くなり、選択肢が**同じ系統(同ジャンル・近い年代)**で固められて難しくなる。

### アイテム(強化)

| アイコン | 名前 | 効果 |
|---|---|---|
| 🐢 | スロー再生 | モザイクが解けるのが25%ゆっくりに(考える時間UP・累積) |
| ㊙ | 予備の㊙ | ㊙を1回復し、最大値も+1(累積) |
| 💰 | 高得点ブースト | 獲得スコアが25%アップ(累積) |
| 💡 | カンペ | 毎ラウンド、不正解の選択肢を1つ自動で消す |
| 🔥 | コンボ番長 | 連続正解の倍率の伸びが1.5倍に(累積) |
| 🛡️ | お守り | 間違えても㊙を失わない権利を2回ぶんチャージ(累積) |

## 作成の背景

「当てる／推理する」系の1人用Webゲームの近年のトレンドを `deep-research`(複数ソースをファンアウト検索し出典付きで検証)で調査し、出典を確認できた事実に基づいて企画した。

- **画像を段階開示して当てる**フォーマットが定番化している。映画を6フレームから当てる [Framed](https://framed.wtf/)、同系の [Flickle](https://www.flickle.co/)(6回挑戦・3フレーム目で公開年、4フレーム目でジャンルを自動ヒント)が代表例。本作の「モザイクが徐々に解ける=情報が段階開示される」コアはこの系譜。
- そのものズバリ、**ミュージックビデオを画像から当てる** [MusicVidle](https://www.musicvidle.com/)(最大6枚の画像でMVを当てる)が存在し、音楽×画像当ての面白さが実証されている。本作はこれを「**モザイク解除 × 早押し4択 × ローグライト**」に再構成し、さらに**正解後に本物のMVを再生**する報酬を足した。
- リテンションは [Wordle](https://arxiv.org/pdf/2305.03502) 型(全員同じ問題を1日1回)や、[Higher or Lower](https://www.higherorlowergame.com/youtube/video-views/) 型(連勝スコア＆ハイスコア更新)が王道。本作は後者寄りの**エンドレス連勝＋自己ベスト更新**を採用し、難易度カーブとアイテム強化で「もう1問」を誘発する。
- 実装基盤は [YouTube IFrame Player API](https://developers.google.com/youtube/iframe_api_reference)。`https://www.youtube.com/iframe_api` を読み込むだけで**APIキー不要・バックエンド不要**、単一HTMLで成立する。

## 技術構成

- **単一HTMLファイル**。外部ライブラリはGoogle Fonts(`Anton`/`Orbitron`/`Zen Maru Gothic`)とYouTube IFrame APIのみ。
- **出題のクリップ画像**: YouTubeのサムネイル `https://i.ytimg.com/vi/<id>/hqdefault.jpg`(480×360・常時存在・CORS `*`)を `crossOrigin="anonymous"` で読み込み、`<canvas>` でモザイク化。小さなオフスクリーンに縮小→`imageSmoothingEnabled=false` で拡大することでドット化し、ブロック数を時間で増やして「解ける」演出にしている。16:9のレターボックス(上下の黒帯)を避けるため中央 480×270 を切り出して描画。
- **正解後の本編**: IFrame APIの `loadVideoById({videoId, startSeconds})` で実際のMVを再生(サビ目安の秒数から)。
- **収録曲は全曲 oEmbed で実在＆埋め込み可能を事前確認**(`https://www.youtube.com/oembed?...`)。世界的ヒットのVEVO/公式チャンネル中心で、地域・期間に強い globally-embeddable な曲を選定。万一再生不可でも `onError` で破綻しないようにしている。
- **iOS消音スイッチ対策**: 効果音(Web Audioのリアルタイム合成)は無音WAVを`<audio>`でループ再生してオーディオセッションを再生カテゴリへ切替＋`AudioContext.resume()`完了待ち＋`interrupted`からの自動復帰(`games/hakai/` の手法を踏襲)。MV本編の音は埋め込みプレーヤー側で鳴る。
- **スコア・最高到達レベルは localStorage** に保存(try/catchでフォールバック)。
- **メインループ(rAF)はスタート画面から常時稼働**し、参照する状態は宣言時に初期化(初回フレームの例外でループが死ぬのを防ぐ)。
- **モバイル対応**: `viewport-fit=cover` ＋ セーフエリア、`touch-action`、ダブルタップズーム無効化(350ms以内の連続`touchend`＋`gesturestart`/`dblclick`の`preventDefault`)、Pointer Eventsで統一、`max-width`の縦カラムで中央寄せ、PC用にキー操作(`1`〜`4`・Space)も用意。

### YouTube埋め込み規約への配慮

YouTubeの埋め込みプレーヤーは、**前面へのオーバーレイ**・**音声と映像の分離**・**再生ボタン以外の操作で動画をゲートすること**を禁じている。そのため本作は——

- **当てる手がかりは「プレーヤー」ではなく公開サムネイル画像**を加工したもの(プレーヤーを覆ったり、音だけ抜き出したりしない)。
- **MVを表示している間は、走査線・タグ・判定表示など一切のUIをプレーヤーの前面に重ねない**(`#stage.clean` で全て非表示にし、判定文は盤外の見出し行へ出す)。プレーヤーは標準コントロール付きで素のまま再生する。

という設計にした。

## 実施したテスト

1. **構文チェック**: インラインJSを抽出して `node --check`(OK)。
2. **実ブラウザE2E(puppeteer-core + ローカルChrome、モバイルビューポート390×780・`hasTouch`)を、`file://`ではなく**ローカルHTTPサーバ経由**で実施**:
   - スタート→出題→正解→リビール→次へ、を複数ラウンド通し、スコアがコンボ倍率込みで増加することを確認(745→1013→…と伸長)。
   - 5問正解でアイテム選択画面が出てピック後に続行、不正解で㊙が減り0でゲームオーバー画面が出ることを確認。
   - `<canvas>` に描画ピクセルが存在(getImageDataで非黒ピクセル多数)、リビールで実際のMV(Blank Space / Smells Like Teen Spirit 等)が埋め込み再生されることをスクリーンショットで目視確認。
   - `pageerror` ゼロ。コンソールエラーは `favicon.ico` の404のみ(サイト共通・無害)。
3. **見た目の目視確認**: スタート／出題(モザイク)／リビール／アイテムの各画面をスクリーンショットで確認。

## 開発で得た教訓

- **YouTube埋め込みは `file://` では再生できない**(`エラー153 動画プレーヤーの設定エラー`)。テストもGitHub Pagesも**http(s)オリジン必須**。`python3 -m http.server` 経由でE2Eした。
- **埋め込みプレーヤーは曲名(タイトル)を常に上部に表示する**ため、「動画を流して当てさせる」案はタイトルでネタバレして成立しない。さらに規約上プレーヤーを覆って隠すこともできない——この2点から「**当てる手がかりはサムネ画像、本編は正解後に素で再生**」という構成に確定した。
- **oEmbedはブラウザ`fetch`だとCORSで弾かれる**(`access-control-allow-origin`なし)。一方サムネ画像は `access-control-allow-origin: *` でcanvas加工可。よって曲名・正解データは**自前で内蔵**し、実行時はサムネ画像とIFrameのみに依存する設計にした。収録曲の実在確認だけはサーバ側(curl)でoEmbedを使った。
- `hqdefault.jpg` は全動画で安定して存在するが、`maxresdefault`/`sddefault` は404があり、`1/2/3.jpg`(絵コンテ枠)は120×90と小さく内容も近い。**当てに使える安定画像は実質 hqdefault 1枚**だったため、複数フレーム式ではなく単一画像のモザイク解除式にした。

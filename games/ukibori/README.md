# ウキボリ（UKIBORI） — サムネ・ピクロス

**動画サムネから作ったお絵かきロジック（ピクロス／ノノグラム）を解くと、暗室の写真が現像されるように本物のサムネが浮き彫りになる。** 行・列の数字の手がかりを頼りにマスを塗り、正しく解けると実在MVのサムネが立ち上がって、その場で**本編を再生**できる。盤面は解くほど大きくなり、制限時間内に解き続けてスコアを伸ばすスコアアタック。

- **プレイURL**: https://nrtkumi.github.io/games/ukibori/
- **対応環境**: スマホ（縦画面・タッチ／ドラッグで連続塗り）ファースト。PCのマウスにも対応。単一HTML。
- **通信**: 必要（YouTubeのサムネ画像とIFrame Playerをブラウザから直接利用）
- **デザインコンセプト**: 「暗室の現像」。深い青みの闇にシアンのセーフライト、見出しはドット風フォント DotGothic16。解けると写真がじわっと現れる“現像”演出。

## 遊び方

1. 行（左）と列（上）の**数字**は、その列で**連続して塗るマスの数**を表す（ノノグラムの手がかり）。
2. **ぬる**モードでマスをタップ／ドラッグして塗る。確信が持てないマスは**✕マーク**で「空き」を覚えておける。
3. すべての行・列の手がかりを満たすと**現像成功**。本物のサムネが浮かび上がる。
4. **▶ 本編を見る**で動画を再生、**次の問題 →**で次へ。盤面は 8×8 → 13×13 へ拡大。
5. **制限時間**を使い切るとゲームオーバー。速く解くほど・連続で解くほど高得点。ベストは保存。
6. **💡ヒント**は正しいマスを1つ自動で埋める（時間を少し消費・回数制限あり）。

## 作成の背景

YouTubeのサムネは `getImageData` 可能な唯一安定した加工対象（[[youtube-embed-gotchas]]）。これを「**ピクセルに落として手がかりに変換**」すれば、サムネそのものがパズルの設計図になる。ノノグラムは「数字に従ってマスを塗る」が直感的で、**解いた瞬間に元画像が現れる**カタルシスが題材と噛み合う。解像度を上げていく構成で、短時間でも達成感が連鎖する。報酬は本編再生——ここでもサムネが手がかり・本編は素の再生というYouTube埋め込みの正攻法を守っている。

## 技術構成

- 単一HTML＋CDN（Google Fonts: DotGothic16 / Noto Sans JP、YouTube IFrame API）。盤面・手がかり・現像は`<canvas>`描画。
- 問題生成: サムネ（`i.ytimg.com/.../hqdefault.jpg` を `crossOrigin="anonymous"`）の中央正方形を `GRID×GRID` に縮小して `getImageData`、輝度を取り、**適応しきい値（約46パーセンタイル）**で白黒の正解パターンを作る。そこから行・列のラン長（手がかり）を算出。
- 勝利判定は**標準ノノグラム準拠**（各行・各列が手がかりを満たせば成立）。自動生成は一意解とは限らないため、「正解パターン完全一致」ではなく「手がかり充足」で判定し、別解でもクリアできるようにした。現像で見せるのは常に本物の写真。
- 入力はPointer Eventsでドラッグ連続塗り（最初のマスで塗る/消すを固定）。`click`非依存。
- 報酬再生は IFrame Player API（`loadVideoById`）。プレーヤー前面にオーバーレイしない。
- iOS対策一式（無音WAVアンロック、`resume()`待ち、`touch-action:none`、ダブルタップズーム無効、毎フレーム`setTransform(dpr…)`）。効果音はWeb Audio合成、ベストは localStorage。

## テスト

- **構文チェック**: `node --check`。
- **実ブラウザE2E**（puppeteer-core、モバイルviewport＋タッチ）: ローカルHTTPサーバ経由で起動→スタート→（テストフックで）正解充足→現像→次問題まで通し、`pageerror`ゼロ、`getImageData`が**サムネ由来**（汚染なし）でパターン生成できること、スコア加算・盤面拡大・本編プレーヤーが**正しい動画IDで**ロードされることを確認。

### 教訓

- レベルアップで`GRID`を同期更新したのに、`cellState`/手がかり配列は**サムネの非同期ロード後に再構築**していたため、読み込み中フレームの描画が旧配列を超参照して例外に。**ロード中は盤面描画をスキップ**して解消。
- 自動生成ノノグラムは一意解とは限らない。「正解と完全一致」を勝利条件にすると、手がかりを全部満たしているのにクリアにならない不可解な状態が起こり得る。**手がかり充足での判定**に切り替えるのが正解。
- `file://` だと同梱・外部画像で `getImageData` が SecurityError になるため、テストは必ずローカルHTTPサーバ経由で。

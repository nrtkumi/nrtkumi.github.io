# バズリ（BAZURI） — 再生回数マージ

**サムネをスワイプで寄せて、同じ動画どうしを合体。** 再生回数の階段を一段ずつ駆け上がり、世界一バズったMV「ベイビーシャーク（約160億回）」を目指す2048系マージ。実在する有名MVのサムネ画像をタイルに使い、合体で到達した動画は**殿堂入り**して、その場で**本編を再生**できる。

- **プレイURL**: https://nrtkumi.github.io/games/bazuri/
- **対応環境**: スマホ（縦画面・スワイプ）ファースト。PC（矢印キー／WASD、Zで戻す）対応。単一HTML。
- **通信**: 必要（YouTubeのサムネ画像とIFrame Playerをブラウザから直接利用）
- **デザインコンセプト**: 「深夜のレコメンド地獄／バズの祭壇」。ほぼ黒の背景にYouTube由来の赤を差し色、再生回数はアンバーの極太コンデンス数字（Anton）。紫グラデやネオングローは使わない。

## 遊び方

1. 盤面を**スワイプ**（PCは矢印キー）すると、全タイルがその方向へ寄る。
2. **同じ動画**のサムネがぶつかると合体し、**一段上の（より再生回数の多い）動画**に進化する。
3. 合体のたびに、その動画の再生回数が「稼いだ再生回数（スコア）」に加算される。
4. 動かせなくなったらゲームオーバー。ベストスコアは保存される。

## 作成の背景

「YouTubeの公開情報・IFrame APIを使ったゲーム」という題から、**サムネ画像＝そのまま手がかりになる素材**である点と、**再生回数という万人が知る指標**を組み合わせた。2048型のマージは「同じものを2つくっつける」が5秒で理解でき、スマホの縦スワイプと相性が良い。タイルを単なる数字ではなく**実在の超有名MV**にして、頂点を「史上最も再生されたMV（ベイビーシャーク）」に置くことで、ゴールが直感的で笑える設計にした。到達報酬を**本編の再生**にすることで、YouTube埋め込みを「ネタバレにならない正攻法」で活かしている。

## 技術構成

- 単一HTML＋CDN（Google Fonts: Anton / Noto Sans JP、YouTube IFrame API）。依存ビルドなし。
- 盤面は`<canvas>`描画。サムネは `https://i.ytimg.com/vi/<id>/hqdefault.jpg` を `crossOrigin="anonymous"` で読み込み（i.ytimg.com は `access-control-allow-origin: *`）、`getImageData`でも汚染されない。レターボックスの中央480×270を切り出してタイルにカバー描画。
- マージは1次元スライド解法（`slideLine`）を4方向に適用。タイルの移動はスライド→ポップのアニメーション。
- 報酬再生は YouTube IFrame Player API（`loadVideoById`）。プレーヤー前面に一切オーバーレイしない（埋め込み規約順守）。サムネが手がかり・本編は素の再生という[[youtube-embed-gotchas]]で確立した構成。
- 収録動画の実在＆埋め込み可否はリリース前に oEmbed（200=実在&embed可）でサーバ側チェック済み。正解データ（再生回数の階段）は自前で内蔵。
- iOS対策: 無音WAVを`<audio>`でループ→消音スイッチ下でも効果音が鳴る、`AudioContext.resume()`待ち、`touch-action:none`、ダブルタップズーム無効、毎フレーム`setTransform(dpr…)`再設定。
- 効果音はWeb Audioでリアルタイム合成。ベストスコアは localStorage。

## テスト

- **構文チェック**: インラインJSを抽出して `node --check`。
- **ロジック単体＋ファズ**: `slideLine`の合体・並びを単体検証し、3000ゲームのランダムプレイで「タイル値が0..10を逸脱しない／例外なし／必ず終局」を確認。
- **実ブラウザE2E**（puppeteer-core＋ローカルChrome、モバイルviewport＋タッチ）: ローカルHTTPサーバ経由でスワイプを連打 → `pageerror`ゼロ、`getImageData`に描画ピクセルあり、スコア／殿堂の解放が進む、殿堂から本編プレーヤーが**正しい動画IDで**ロードされることを確認。

### 教訓

- 勝利演出を「ゲーム終了」にすると、スタートボタンに紐づいた`pointerup`ハンドラと衝突した。頂点到達は終局にせず**祝福トースト＋エンドレス継続**に変更して解消。
- クロスオリジン画像を`getImageData`したい場合は `i.ytimg.com` ＋ `crossOrigin="anonymous"` が必須（これを欠くとcanvasが汚染され実機テストでSecurityError）。

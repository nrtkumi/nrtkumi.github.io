# 花あわせ — 花札コンボ・デッキビルドローグライト

**デザインコンセプト**: 深夜の漆黒卓上——黒漆の卓に金箔の文字、花々の絵札が並ぶ高賭けの花合わせ。

## プレイURL

https://nrtkumi.github.io/games/hanaawase/

## 対応環境

- iOS Safari / Android Chrome (スマホファースト、縦画面)
- PC ブラウザ (中央カラム表示)
- 通信不要、ローカル完結

## 遊び方・ルール

### 基本の流れ

1. **手札(6枚)から役になるカードを選んでタップ**
2. **「あわせる」ボタンで役を成立させ、文(点数)を獲得**
3. `獲得文 = 基礎点 × 倍率` で計算。役が重なるほど倍率が積み上がる
4. 月ごとに設定された**目標文**に到達すれば次の月へ
5. 手番(4回)・捨て回数(3回)を使い切る前に目標に届かなければゲームオーバー
6. **道具市**で「お守り」を購入し、得点倍率や手番数を強化して次の月に備える
7. 12月(十二月)を制覇すれば完全制覇

## 作成の背景

### ローグライトデッキビルダーが最強の「もう一回」を生む

花あわせは Balatro ([Wikipedia](https://en.wikipedia.org/wiki/Balatro)) のゲームデザイン哲学を花札に移植したものだ。Balatro が証明したのは「コンボのシナジー × アンテの段階的目標 × ショップでのメタ進行」という三角形が、ゲーマーを最も長く席に留める構造だということ。

ローグライクゲームの成功要因の分析 ([Harold Thomas, Medium](https://medium.com/@hrldthomas/deconstructing-indie-success-a-growth-pms-guide-to-idle-and-roguelike-games-6a2f982aacaf)) によれば、addictionを生む鍵は「reward-prediction-error」——予想外に大きなコンボが炸裂した瞬間の驚き——であり、これがリプレイ衝動を作る。花あわせでは `60文 × 4倍 = 240文` といった倍率スタックの表示をあえて分解して見せることで、このドーパミンスパイクを狙っている。

### スマートフォンのサムゾーン設計

Smashing Magazine の親指ゾーン研究 ([Smashing Magazine](https://www.smashingmagazine.com/2020/02/design-mobile-apps-one-hand-usage/)) に従い、すべての操作ボタン(あわせる/捨てる)を画面下部のサムゾーンに配置。手札カードもその直上に置き、片手で完結する操作系を実現している。

## 技術構成

- **単一HTML**（CSS/JS インライン、外部アセットなし）
- **DOM ベースのカード UI**（canvas 不使用）
- **Web Audio API**（効果音リアルタイム合成、外部音源なし）
- **iOS 音声対策**: 無音WAV `<audio>` ループ + `AudioContext.resume()` の Promise 完了待機
- **ダブルタップズーム防止**: `touchend` 350ms ガード + `gesturestart` preventDefault
- **入力**: Pointer Events のみ（click 不使用）
- **永続化**: localStorage（壊れた場合はフレッシュスタートにフォールバック）
- **フォント**: Shippori Mincho B1 / Zen Maru Gothic (Google Fonts CDN)

# 影法師一座 — KAGEBOSHI

切り絵の役者を**舞台(グリッド)**に並べ、隣り合わせで力を響かせる。幕が上がれば、あとは芝居=自動戦闘まかせ。勝てば次の宿場へ巡業する、**影絵芝居のオートバトラー・ローグライト**。

- プレイ: https://nrtkumi.github.io/games/kageboshi/
- 対応: スマホ(タッチ)/PC(マウス・ドラッグ、Space/Enterで開幕、1・2・4で倍速)。縦画面・単一HTML。

## 遊び方

1. **支度** — 下の「控え」や「旅芸人の店」から、役者と小道具を**舞台へドラッグ**して配置する。舞台に乗った役者だけが戦う。役者を売る時はドラッグ中に現れる「売却」枠へ。
2. **隣接シナジー** — 駒は上下左右の隣と力を響かせる。太鼓は隣の手数を速め、提灯は隣の攻を上げ、鈴は隣の妖を強くする…並べ方が一番の腕の見せどころ。
3. **幕を上げる** — タイムライン式の自動戦闘。役者は「間(ま)」ごとに動き、相手の座(気力)を先に削りきれば勝ち。×1/×2/×4で観戦速度を変えられる。
4. **巡業** — 勝てば銭を得て次の宿場へ。負けると命が1減り、命(♥×3)が尽きると一座は解散。全10宿場、最後は大物の怪「大百足」との大舞台。

### 小道具(プロップ＝隣接バフ)

太鼓(手数×0.82) / 提灯(攻+2) / 御幣(行動で座+2回復) / 鈴(隣の妖の攻+4) / 面(隣の鬼・獣の攻+3) / 屏風(2秒ごと守り+9) / 毛氈(隣接ぶん最大気力+12)

## 作成の背景

2026年の App Store / Google Play・Steam の市場調査(deep-research で出典付き検証)から、**インベントリ/編成型オートバトラー**が1人用の実証済みコアループとして転用価値が高いと判明した。代表例は *Backpack Battles*(グリッドに装備を配置しシナジーを組む、戦闘は自動・初月64万本)や、ローグライト・デッキビルダーの大潮流(*Balatro* / *Slay the Spire*)。当サイトの既存45本にオートバトラーは存在せず、ジャンルの空白だったため本作を制作した。

- インベントリ型オートバトラー: https://store.steampowered.com/app/2071230/Backpack_Battles/
- ローグライトの潮流(Balatro 等): https://www.gamesradar.com/games/roguelike/this-slick-roguelike-deckbuilder-is-shoulder-to-shoulder-with-balatro-and-slay-the-spire-on-steam-top-sellers-and-its-dev-is-feeling-the-pressure-the-definition-of-walking-among-giants/

「隣接で強くなる空間パズル + 観るだけの自動戦闘 + 巡業ローグライト」を、スマホのドラッグ操作だけで完結する規模に落とし込んだ。

## デザインコンセプト

**影絵芝居 / 切り絵**。夜の障子を後ろから照らした和紙のスクリーン、木枠の舞台、吊り提灯と紙の月。役者は種族ごとに色分けした切り絵のシルエット(canvasのパスで描画＋落ち影)。配色は藍墨色の夜・生成りの灯り・朱・金。フォントは見出しに筆書きの Yuji Syuku、小見出しに Shippori Mincho。生成AI的な紫グラデやネオンは排した。

## 技術構成

- 単一HTML・依存ライブラリなし。canvas 2D による手描きの切り絵シルエットと舞台演出。
- 戦闘はタイムライン式の自動解決(各アクターが cad 周期で行動／呪い=DoT・守り=ブロック・回復・隣接バフを毎フレーム処理)。
- iOS消音スイッチ対策(無音WAVループ＋AudioContext復帰)、ダブルタップズーム無効化、セーフエリア対応、PCワイド画面で縦カラムに制限。効果音はWeb Audioのリアルタイム合成。
- 進行/最高到達は localStorage に保存。

### バランス調整(ボットシミュレーション)

戦闘ロジックを Node に移植し、「限界効用で配置を選ぶ skilled-bot」で各 4000〜1500 ラン回して難易度曲線を調整。最終的に**ボットの全10宿場クリア率 ≈ 47%**、敗北のピークは大百足戦(ラスボス)というカーブに収束させた(序盤〜中盤は学習帯、第7・第9宿場に小さな山場)。完全情報ボットは実プレイよりやや甘く出る前提で、ラスボスを「初見では半々、編成を詰めれば勝てる」山場として設定。

## 実施したテスト

- インラインJSを抽出し `node --check`(構文OK)。
- Node 上での戦闘バランスシミュレーション(配置最適化ボット、計 ~1.5万ラン)。
- 実ブラウザE2E(puppeteer-core + ローカルChrome, iPhoneビューポート/タッチ): 開始→店で購入→控えから舞台へドラッグ配置→開幕→自動戦闘→決着→結果表示までを通し、pageerror ゼロ・canvas描画あり・倍速操作・各画面のスクリーンショット目視を確認。`file://` ではなくローカルHTTPサーバ経由で実施。

## 教訓

- オートバトラーは**配置=空間パズルが面白さの核**なので、隣接バフの種類を「速さ・攻・回復・守り・最大HP・種族特化」と役割で散らすと編成の幅が出る。
- 自動戦闘は観戦が単調になりがち。倍速ボタン・ダメージ/回復のフロート・幕/月/提灯の舞台演出で「芝居を観ている」感を補強した。
- ラスボスの強さは手計算では詰めきれず、戦闘ロジックをそのまま Node に移植したボットシミュレーションで初めて妥当なカーブに落ちた。

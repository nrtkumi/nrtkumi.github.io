# いつの絵? (ITSU NO E?)

シカゴ美術館(Art Institute of Chicago)に実在するパブリックドメインの名画を見て、**「何年の作品か」を年代スライダーで当てる**推理ゲーム。制作年に近いほど高得点(GeoGuessr式スコアリング)。全8作品、満点8000点。

- **プレイURL**: https://nrtkumi.github.io/games/itsunoe/
- **対応環境**: スマホ(縦画面・タッチ)/ PC(マウス+←→キー微調整)
- **通信**: 必要(Art Institute of Chicago の公開APIと画像をブラウザから直接取得)

## 遊び方

1. 額縁の中に名画が表示される
2. 年代スライダー(1300〜1950年)をドラッグして「これだ!」と思う制作年に合わせる(−1/+1ボタンや←→キーで微調整、Shift+←→で±10年)
3. 「これだ!」で確定。スライダー上に正解の年代が金のマーカーで現れ、予想とのズレが点線で結ばれる
4. 作品名・作者・年代表記(例: 1631–34)と獲得点を確認して次の作品へ。8作品の合計がスコア

| 要素 | 内容 |
|---|---|
| 採点 | 制作期間(date_start〜date_end)に的中で**満点1000点**。外れたら年差に応じて指数的に減点: `round(1000 × exp(−年差/25))`(1年ズレ961点、25年ズレ368点、100年ズレ18点) |
| ランク | 合計点で判定: 美術館めぐり初心者 → 学芸員見習い(2000)→ 学芸員(3500)→ 鑑定士(5000)→ 大鑑定士(6200)→ 伝説の大鑑定士(7200) |
| ベスト | 自己ベストを端末に保存(localStorage) |
| シェア | リザルトから X(Twitter)にスコアとランクをシェア |

## 作成の背景

市場調査の要点:

- 2026年のブラウザゲームはハイパーカジュアルがプレイ時間シェア78.8%を占め、「最初の10秒でルールが分かる」ことがヒットの条件([primepulselogic](https://primepulselogic.com/browser-games-2026-viral-trend/))
- 検索トレンドでも Block Blast・spacebar clicker など「説明不要の即遊べる系」が急伸([Rising Trends](https://www.risingtrends.co/trends/gaming-trends-2026))
- ブラウザゲームシーンはNYT Games系のデイリー/クイズ系とindie HTML5が主流([Dinogame GG](https://dinogame.gg/blog/best-new-browser-games-2026/))

GeoGuessr が確立し、Timeguessr・Wordle系デイリーゲームへと続く**「推理して、正解との距離で採点される」**型は、運ではなく知識と観察眼が点数に直結するのが中毒性の核。これを「場所」ではなく「名画の制作年」に適用した。服装・画風・額装から年代を推理する体験は美術の知識がなくても直感で楽しめ、リビールで必ず「作品名と作者を知る」学びが残る。サイト内の既存ゲーム(アクション、マージ、比較クイズ、音楽クイズ、サバイバー等)とジャンルが被らない「鑑定スライダー」枠。

## 技術構成

- 依存ライブラリなしの単一HTML(DOMベース、canvas不使用)
- **使用API(キー不要・無料・CORS対応・データはCC0)**
  - 作品検索: `https://api.artic.edu/api/v1/artworks/search` に `query[bool][must][][term][is_public_domain]=true` + `query[bool][must][][term][artwork_type_id]=1`(絵画)のboolクエリ。人気順(関連度スコア順)で返るため有名作から出題される
  - 画像(IIIF): `https://www.artic.edu/iiif/2/{image_id}/full/600,/0/default.jpg`
- **出題プールの設計**(実APIの分布を事前調査して決定): searchエンドポイントはoffset上限1000件(100件×10ページ)。絵画フィルタ後の全1000件は画像と年代が揃っており、年代分布は1300〜1935年に集中(1800年代が約半数)。そこで採用条件を `image_id あり / date_start・date_end が 1300〜1950 / 制作期間60年以内` とし、スライダー範囲を1300〜1950年に設定。10ページをシャッフルして1ページずつ取得し、ページ内もシャッフルして出題
- 画像ロード失敗時はその作品をスキップして次の候補を出す(タイムアウト12秒、最大80回リトライ)。次ラウンドの作品+画像はリビール中に裏でプリフェッチして待ち時間ゼロ
- スライダーはPointer Eventsの自前実装(`setPointerCapture`でドラッグ、−1/+1ボタンは長押しで連続調整、PCは←→キー)
- 効果音はWeb Audioでリアルタイム合成(的中はアルペジオ、外れは沈む低音など上品なトーンで統一)。iOS消音スイッチ対策(無音WAVを`<audio>`でループ再生してオーディオセッションを切り替え、`AudioContext.resume()`のPromise完了待ち、`interrupted`からの自動復帰)
- 入力はPointer Eventsで統一(`click`不使用)、ダブルタップズーム無効化3点セット、PCワイド画面は縦長カラム(`min(100vw, 520px, 64vh)`)に制限
- ベストスコア・ミュート設定はlocalStorage(try/catchでフォールバック)
- OGP画像はcodex CLI経由のgpt-image-2で生成(金の額縁の名画+大きな「?」+タイトルロゴ、ダークな美術館壁面の構図を日本語で指示。1536x1024 → sipsで1200x800 JPEGに変換)
- クレジット表記: タイトル画面に「作品データ・画像: Art Institute of Chicago (CC0 / Public Domain)」
- E2E用フック: `window.IE`(phase / round / score / guessYear / lastPoints / current 等の読み取り専用getter)

## 開発で得た教訓

- AIC searchエンドポイントは **offset上限1000件**(超えると `403 Invalid number of results`)。ランダムページサンプリングは10ページの範囲内で行う必要がある。一方で関連度順=人気順なので、上限内の1000件はちょうど「誰でも見たことがありそうな名画」プールとして機能する
- `www.artic.edu` の画像CDN(IIIF)はCloudflareのbot判定があり、**curlや自動操縦のChromeには403チャレンジを返す**(通常のブラウザは素通り)。E2Eテストでは先にIIIF画像URLへ直接アクセスしてチャレンジを解かせ、`cf_clearance`を取得してからゲームをロードする手順が必要だった
- 年代スライダーは650年を画面幅~326pxに割り当てるため1pxが約2年。タッチだけで1年単位を合わせ込むのは不可能なので、**ドラッグ(粗調整)+長押し対応の−1/+1ボタン(微調整)**の2段構えにした

# PK魂 (PK TAMASHII)

PK戦だけで世界の頂点を決める5連戦ノックアウトトーナメント。
攻撃は「ドラッグで狙い、揺れる照準が静まる瞬間に離してシュート」、守備は「相手の視線を読み、ゴールをタップして横っ飛びセーブ」。勝ち抜くたびに強化を1つ選ぶローグライト風サッカーゲーム。

- **プレイURL**: https://nrtkumi.github.io/games/pk-tamashii/
- **対応環境**: スマホ(タッチ操作・縦画面)/ PC(マウス+キーボード)

## 遊び方

- ベスト32 → ベスト16 → 準々決勝 → 準決勝 → 決勝 の5試合をPK戦(5人制+サドンデス)で勝ち抜くと優勝。
- **自分のキック**: 画面をドラッグすると照準がゴール内を動く。照準は常に揺れており、**揺れが静まって狙い通りの位置に来た瞬間**に指を離して蹴る。コーナーぎりぎりはキーパーに届きにくいが、揺れで枠を外すリスクがある。
- **相手のキック**: 助走中に光る矢印は相手キッカーの視線。ただし**ウソの場合もある**(終盤のラウンドほど信用できない)。ゴール内をタップするとその位置へ横っ飛び。**早くタップするほど遠くまで届く**が、ボールを見てから飛ぶと届く範囲が狭まる。
- 4本目以降とサドンデスは**プレッシャー**で照準の揺れが激しくなる。
- PC: ←→↑↓で照準移動/Spaceでシュート。守備はQ/E(上隅)・A/D(左右)・S(中央)など。

### 強化(試合に勝つたび3択から1つ)

| 強化 | 効果 |
|---|---|
| 🎯 精密キック | 照準の揺れ -25%(2段階) |
| ❤️‍🔥 鋼の心臓 | プレッシャーで揺れが増えない |
| 👁️ 千里眼 | 相手の視線(矢印)が正直になりやすい(2段階) |
| 🧤 神の手 | セーブできる範囲 +20%(2段階) |
| 🐢 ゾーン | 相手のシュートがスローに見える |
| 🌀 魔球カーブ | 敵キーパーにコースを読まれにくい(2段階) |
| 🔥 威圧オーラ | 相手キッカーが枠を外しやすくなる(2段階) |
| 🔁 もう一蹴り | 1試合に1回、失敗キックをやり直す |

## 作成の背景(市場調査メモ)

2026年は北中米ワールドカップ開催年で、サッカー題材の旬に合わせて企画。市場調査の要点:

- 2026年のカジュアルゲーム市場では「5秒で理解できる直感的メカニクス」が引き続きバイラルの条件とされ、ソロトーナメントなど**競技イベント形式**と、シンプルな核に成長・収集を重ねる**ハイブリッドカジュアル**が主流([Refybe](https://refybe.com/en/blog/2/hyper-casual-games-trends-and-the-future)、[Udonis](https://www.blog.udonis.co/mobile-marketing/mobile-games/casual-games)、[Game Growth Advisor](https://gamegrowthadvisor.com/blog/2026-04-16-hybrid-casual-game-design-strategy-2026/))。
- Web圏のサッカーゲームはPK・ペナルティシュートアウトものが定番人気で、トーナメントを勝ち上がる形式や、方向・タイミングのメーター操作が王道([Poki World Cup特集](https://poki.com/en/world-cup)、[CrazyGames World Cup Penalty](https://www.crazygames.com/game/world-cup-penalty)、[COKOGAMES](https://www.cokogames.com/penalty-kicks-world-cup-game/))。
- 本作はPKの「読み合い」を核に、トレンドのトーナメント形式+ローグライト強化(うんだめし・HAKAIと同系の試合間3択)を重ねた。既存ゲームと被らない初のスポーツ枠。

## 技術構成

- 依存ライブラリなしの単一HTML(canvas + Pointer Events)。ゲーム内サイズは論理幅400px固定で、`max-width: min(520px, 64vh)`の縦長カラムをPCでも中央表示。
- 効果音はWeb Audioでリアルタイム合成(ホイッスル・キック・ネット・ポスト・歓声・心音)。観客のざわめきはループするブラウンノイズ+LFO。iOSの消音スイッチ対策として初回タップで無音WAVを`<audio>`ループ再生し、`AudioContext.resume()`のPromise完了を待ってアンロック。
- ダブルタップズーム無効化3点セット、`touch-action`制御、セーフエリア対応。dpr変換は毎フレーム`ctx.setTransform`で再設定。
- 戦績(優勝回数・最高成績)はlocalStorage(try/catchフォールバック)。
- 画像アセットはcodex CLI経由のgpt-image-2で生成(夜のスタジアム観客席背景・「PK魂」キービジュアル)。スタジアム画像は読み込み失敗時にcanvas手描きスタンドへフォールバック。OGPはキービジュアルを1200x630にクロップ。

## 実施したテスト

1. `node --check` によるインラインスクリプトの構文チェック
2. ヘッドレスロジックテスト34項目(DOMスタブ+rigフックで決定論化): ゴール/セーブ/反射セーブ/枠外/ポスト判定、3-0早期決着、5-5からのサドンデス、強化適用、敗北時の戦績保存など
3. 実ブラウザE2E 24項目(puppeteer-core + ローカルChrome、ローカルHTTP経由): モバイルビューポート(`hasTouch`)でタッチドラッグシュート・タップダイブ・強化カードタップ・リトライ・ミュートを実操作、canvas描画ピクセル確認、console/pageエラーゼロ、デスクトップのキーボード操作(矢印+Space)も検証

### 開発で得た教訓

- rigフック(キーパーの行動や相手の狙いを固定するテスト用フック)は**どのタイミングで読まれるか**が重要。相手キックのパラメータはkeepWait突入時に確定するため、テスト側はその前にrigを設定する必要があった(テスト失敗の原因は全部これだった)。
- 「揺れる照準を任意のタイミングで離す」は、タイミングゲーの緊張感とエイムの自由度を1ジェスチャーで両立できる。プレッシャー演出(揺れ増加+心音)とも相性が良い。

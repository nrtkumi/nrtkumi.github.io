# ドアップ! (DOAPPU!)

超ドアップにズームされた**実在の生き物写真**が約8秒かけてゆっくりズームアウト。
正体がわかった瞬間に4択から名前をタップする早押しクイズ。早く当てるほど高得点(100〜1000pt)。
出題は iNaturalist の研究グレード観察写真から毎回ランダムに変わる。

- **プレイURL**: https://nrtkumi.github.io/games/doappu/
- **対応環境**: スマホ(縦画面・タッチ)/ PC(1〜4キー・マウス)
- **通信**: 必要(iNaturalist の公開APIをブラウザから直接叩く)

## 遊び方

1. ラウンド開始時に「つぎは…🐦 鳥!」のようにカテゴリが予告される
2. 超ドアップ(8倍)の写真が約8秒かけてゆっくりズームアウトしていく
3. 正体がわかったら下の4択から名前をタップ。**残りズーム率が大きいほど高得点**
4. 答え合わせで写真全体+和名+学名+撮影者クレジットを表示。全10ラウンド・ライフ3

| 要素 | 内容 |
|---|---|
| スコア | 回答時点の残りズーム率に応じて 100〜1000pt(ズームアウト完了後でも回答可・最低100pt) |
| コンボ | 連続正解で +50pt × 連続数 のボーナス |
| ライフ | 不正解で ❤️ -1(正解は表示される)。ライフ0で終了 |
| カテゴリ | 鳥・哺乳類・昆虫・爬虫類・両生類・魚をラウンドごとに切り替え |
| ダミー選択肢 | 同じカテゴリの別の生き物の和名(似たもの同士で迷わせる) |
| リザルト | スコア・正解数・ベスト保存(localStorage)・X シェア |

## 作成の背景

市場調査の要点:

- 2026年のブラウザゲームはハイパーカジュアルがプレイ時間シェア78.8%を占め、「最初の10秒でルールが分かる」ことがヒットの条件([primepulselogic](https://primepulselogic.com/browser-games-2026-viral-trend/))
- 検索トレンドでも Block Blast・spacebar clicker など「説明不要の即遊べる系」が急伸([Rising Trends](https://www.risingtrends.co/trends/gaming-trends-2026))
- ブラウザゲームシーンはNYT Games系のデイリー/クイズ系とindie HTML5が主流([Dinogame GG](https://dinogame.gg/blog/best-new-browser-games-2026/))

「画像の段階的リビール」はテレビのズームクイズ番組や、Wordle系デイリーパズルの系譜(1枚の画像が少しずつ明かされる Framed / Pokedoku 的な型)で実績のあるフォーマット。そこに「いつ答えるか=スコアを賭けたチキンレース」という早押しのリスクリターンを足したのがコア体験。実在の観察写真を使うため、出題が毎回変わってコンテンツが古びない。サイト内の既存ゲーム(アクション、マージパズル、比較クイズ、サバイバー等)と被らないリビール型クイズ枠。

## 技術構成

- 依存ライブラリなしの単一HTML(DOMベース、写真のズームはCSS transform + rAF駆動)
- **使用API(キー不要・無料・CORS対応)**: iNaturalist `GET /v1/observations?quality_grade=research&photos=true&iconic_taxa={分類}&per_page=50&page={1〜15ランダム}&locale=ja&photo_license=cc0,cc-by,cc-by-nc&popular=true`
  - `locale=ja` で `taxon.preferred_common_name` に和名が入る(和名未登録の観察は出題から除外、同名重複も除外)。実測の和名ヒット率は鳥45/50、哺乳類38/50、両生類34/50、魚17/50、爬虫類16/50、昆虫13/50で、1ラウンドに必要な4種(正解+ダミー3)は全カテゴリで余裕をもって確保できる
  - レスポンスが重い(50件で数百KB)ので取得後すぐ `{和名, 学名, 写真URL, attribution}` だけに絞って保持
  - 写真は `square.jpg` → `medium.jpg` 置換で500px版を取得。`Image()` でプリロードし、ロード失敗した観察はスキップして次の候補へ
- ズーム演出: `scale(8)`→`scale(1)` を8秒、`Math.pow(8, 1-p)` で知覚的に等速なズームアウト。`transform-origin` は写真ごとにランダム(12〜88%)で毎回違う部位から始まる
- 次ラウンドはリビール中に裏でプリフェッチ(観察データ取得+画像プリロード)して待ち時間ゼロに。カテゴリごとの候補プールはページをまたいで蓄積・キャッシュ
- 効果音はWeb Audioでリアルタイム合成(ズーム中の加速するチック音、正解ファンファーレ、不正解ブザー)。iOS消音スイッチ対策(無音WAVを`<audio>`でループ再生、`AudioContext.resume()`のPromise完了待ち、`interrupted`からの自動復帰)
- 入力はPointer Eventsで統一(`click`不使用)、ダブルタップズーム無効化3点セット、PCワイド画面は縦長カラム(`min(100vw, 520px, 64vh)`)に制限
- ベストスコア・ミュート設定はlocalStorage(try/catchでフォールバック)
- OGP画像はcodex CLI経由のgpt-image-2で生成(羽根と爬虫類の目のドアップ+虫眼鏡+タイトルロゴの横長キービジュアルを日本語で指示、1672x941→1200x800 JPEGにsipsで変換)
- クレジット表記: タイトル画面に「写真・データ: iNaturalist (CC0 / CC BY / CC BY-NC ほか)」、各ラウンドの答え合わせに撮影者のattribution文字列をそのまま表示

## 実施したテスト

1. **構文チェック**: インラインscript抽出 → `node --check` パス
2. **ヘッドレスロジックテスト(Node、DOM/fetch/rAF/Imageスタブ+合成観察データ)**: 全32チェックパス
   - 和名なし・同名重複の除外、画像ロード失敗時のスキップ
   - 開始→ズーム進行→早押しスコア計算(2秒時点775pt)→不正解でライフ減→ズーム完了後の最低点100pt→コンボボーナス→10ラウンド完走→リザルト→ベスト保存
   - 3連続ミスでGAME OVER、通信エラー表示→再試行で復帰
3. **実ブラウザE2E(puppeteer-core + ローカルChrome、390x844・hasTouch、実API、ローカルHTTPサーバ:8773経由)**: 全18チェックパス
   - 実APIから取得した写真で10ラウンド通しプレイ(タッチ操作のみ、正解9/10・8489pt)
   - 早押しスコア・コンボ・ライフ減・リザルト・ベスト保存・リトライを確認
   - `pageerror`・consoleエラー 0件、スクリーンショットで見た目を目視確認

## 開発で得た教訓

- iNaturalist APIは `locale=ja` を付けるだけで `preferred_common_name` が和名になるが、ヒット率はカテゴリ差が大きい(鳥90%・昆虫26%)。和名なしを除外しても50件/ページあれば4択クイズには十分
- `popular=true`(お気に入りが付いた観察)を付けると「被写体がちゃんと写っている写真」の率が体感で大きく上がる。クイズの成立に効く
- ズームアウトは線形の `scale` 補間だと序盤の縮小が速く感じる。`Math.pow(SCALE_MAX, 1-p)` の幾何補間にすると知覚的に等速で、緊張感が最後まで持続する

# いきもの写真サファリ — 一瞬を、撮れ。

ファインダーを横切る**本物の生き物**を、ピントが合う一瞬にタップして撮影する撮影アクション・ローグライト。題材は世界中のユーザーが撮った実写真(iNaturalist)で、**レアな生き物ほど臆病で逃げ足が速い**。撮った種は図鑑に永久に残る。

- **プレイURL**: https://nrtkumi.github.io/games/safari/
- **対応環境**: スマホ(縦持ち・タッチ)推奨。PC でもプレイ可(スペースキーでピント/撮影)
- **通信**: 必要(iNaturalist の公開APIをブラウザから直接叩く)
- **コアメカニクス(1文)**: 長押しでピントを合わせ、被写体が中央に来た瞬間に指を離して撮影する

## 遊び方

1. **長押しでピント** — 画面を押している間、フォーカスメーターが上昇する。中央のリティクルに最も近い生き物にリングがロックオンする。
2. **緑ゾーンで離して撮影** — メーターが緑(ジャストピント)のときに指を離すと高評価。早すぎると**ピンぼけ**、長く押しすぎると**ボケ**て星が下がる。
3. **構図も採点** — 被写体が中央に近いほど高得点。生き物は横切っていくので、中央に来る一瞬を待つ。
4. **フィルムは有限** — 各エリアの撮影枚数には限りがある。コモンを撮るか、レアを待つか。
5. **逃げる前に撮れ** — レアな個体ほど早く逃走モードに入り、加速して画面外へ消える。逃げ際を撮ると**逃走ボーナス**。
6. 1回の遠征は **4エリア**(もり → みずべ → そら → よる)。エリア間のキャンプで装備を補給し、最後にスコアと図鑑を集計する。

### レア度(実データ由来)

各種のレア度は iNaturalist の**世界の観察数(`observations_count`)から自動算出**している。世界であまり撮られていない生き物ほど高レアで、点が高く、逃げ足も速い。

| レア度 | 観察数の目安 | 性質 |
|---|---|---|
| ★ ふつう | 12万件以上 | のんびり・基礎点低め |
| ★★ ときどき | 4万〜12万 | やや速い |
| ★★★ めずらしい | 8千〜4万 | 速い・小さめで撮りにくい |
| ★★★★ レア | 1500〜8千 | 逃げ足が速い |
| ★★★★★ げきレア | 1500件未満 | すぐ逃げる・高得点 |

### 強化(キャンプ補給)

撮影で得たポイントで、その遠征中ずっと効く装備を買える(図鑑の進捗には影響しない)。

| 装備 | 効果 | 最大Lv |
|---|---|---|
| 🎞️ フィルム増量 | 各エリアの撮影枚数 +2 | 4 |
| 🔭 望遠レンズ | 被写体がゆっくり&ロック範囲拡大 | 4 |
| 📐 三脚 | ピントが安定し緑で止めやすい | 4 |
| 🥜 まきえ | 各エリア最初の1体がレア寄りに | 1 |

### 図鑑

撮影した種は**ブラウザに永続保存**され、写真・和名・学名・レア度・ベスト評価・世界の観察数・撮影者クレジットが残る。遠征のたびに新種を探して埋めていく。

## 作成の背景

このサイトには既に40本以上の1人用ゲームがあるが、ジャンルを見渡すと **「高い/低いを当てる」「クイズ」系が大きく偏って多い**(国・記事閲覧数・為替・絵画の年代・楽曲当て 等)。そこで今回は意図的に、サイトに無い**「撮影アクション × コレクション・ローグライト」**というジャンルを狙った。

設計の軸は2つ:

- **実APIをクイズの答えではなく"ゲームの中身"そのものに使う**。生き物の写真・和名・学名はもちろん、**レア度や逃げ足の速さまで実データ(世界の観察数)から導出**することで、「現実で珍しい個体ほどゲームでも捕まえにくい」という、データと手触りが直結した手応えを作った。
- **写真ゲーム × トップトランプ的なレア度収集**という枯れて面白い組み合わせを、スマホのワンジェスチャ(長押し→離す)に落とし込む。『New Pokémon Snap』的な"撮る楽しさ"と、ローグライトの強化・周回性を、単一HTMLで成立する規模に凝縮した。

データソースには **iNaturalist** の公開 Observations API を採用。キー不要・CORS 対応(`Access-Control-Allow-Origin: *`)で、`research`グレード・`popular`(高評価写真)・CC ライセンス写真に絞れば、和名(`preferred_common_name`)付きの良質な実写真が1リクエストで多数得られる。

- iNaturalist API: https://api.inaturalist.org/v1/docs/
- 写真・観察データのライセンス: CC0 / CC-BY / CC-BY-NC(権利は各観察者に帰属。ゲーム内の撮影画面と図鑑に出典を表示)

## 技術構成

- **単一HTMLファイル**。描画は DOM + CSS(生き物は実写真の `<img>` を絶対配置でアニメーション、ファインダーHUD・ピントリング・フラッシュは CSS/SVG)。canvas を使わないため画像の cross-origin 汚染問題が無く、低負荷で滑らか。
- **デザインコンセプト**: 「70年代の生物図鑑 × フィルム一眼レフのファインダー」。クリーム色の紙、フォレスト/オリーブとバーントオレンジ、黒いファインダーHUDに等幅フォントのEXIF風表示・四隅ブラケット・RECドット。AI感の出る紫グラデやグラスモーフィズムは不使用。フォントは Zen Maru Gothic / BIZ UDPMincho(学名のイタリック)/ Roboto Mono(HUD)。
- **API**: iNaturalist Observations を**エリアごとに1リクエスト**(`iconic_taxa` でバイオーム別、`hrank=species&lrank=species` で種ランクに統一、ランダムページで毎回違う顔ぶれ)。取得した生き物プールからローカルでスポーンする。
- **レア度算出**: `taxon.observations_count` を5段階のティアにマッピングし、基礎点・サイズ・逃走速度・逃走までの時間に反映。
- **フォールバック**: 通信失敗・写真欠損時は絵文字スプライト+オフライン用プールに自動フォールバック(`<img>` の `onerror` 含む)。無言で壊れない。
- **永続化**: 図鑑と自己ベストを localStorage に保存(try/catch フォールバック)。
- **効果音**: 外部音源なし。Web Audio でシャッター音・フォーカス音・撮影成功・新種ファンファーレ等をリアルタイム合成。
- **iOS音声対策**: 初回タップで無音WAVを `<audio>` ループ再生してオーディオセッションを再生カテゴリへ切替、`AudioContext.resume()` の完了待ち、`onstatechange` で `interrupted` から自動復帰。
- **スマホ対応**: viewport-fit=cover + `env(safe-area-inset-*)`、`touch-action`、ダブルタップズーム抑止(`touch-action:manipulation` + `dblclick`/`gesturestart` preventDefault + 350ms以内の連続`touchend`抑止)、入力は Pointer Events 統一で `click` 非依存。PC ワイド画面対策にルート要素を縦長カラム(`max-width: min(100vw, 60vh, 460px)`)へ中央寄せ、キーボード操作対応。
- **メインループ**: rAF はタイトル画面から常時回り、ループが触る状態は全て宣言時に初期化(初回フレーム例外でループが死ぬのを防止)。

## 実施したテストと教訓

- **構文チェック**: インライン script を抽出して `node --check`。
- **実ブラウザ E2E(puppeteer-core + ローカル Chrome、モバイルビューポート+タッチ)**:
  - `pageerror` ゼロを確認(残った 404 は favicon と、iNat 側に実体が無い一部写真のみ。後者は `onerror` で絵文字フォールバックされ、ゲームは無傷)。
  - タイトル→開始→生き物スポーン→長押し撮影→採点→ショップ→4エリア走破→リザルト、までを通しで検証。
  - 位置認識する簡易ボットで全4エリアを走破し、リザルト集計(スコア/撮影数/新種/最大コンボ/図鑑累計)と図鑑・詳細表示・出典表示を確認。
  - 各画面のスクリーンショットを撮って目視確認。
- **得られた教訓**:
  - **`translate(x, y%)` の `%` は要素自身のサイズ基準**で、コンテナ基準ではない。生き物の縦位置を要素相対%で指定していたため全個体が画面上部に張り付くバグが発生 → `left/top` をコンテナ相対%で指定して解消。実ブラウザのスクリーンショット目視で初めて気づいた。
  - iNaturalist は `hrank=species&lrank=species` を付けないと**亜種ランク**が混ざり、和名が欠落し観察数も極端に小さくなる(レア度算出が壊れる)。種ランクに固定すると和名付き・観察数7〜十数万件の良いレンジが得られた。
  - ファインダーの「ロック範囲」と「中央寄せの採点重み」は当初ゆるすぎて、画面端の被写体でも高評価が出てしまった。範囲を狭め構図の重みを上げ、「中央に来る一瞬を待つ」スキルが効くよう調整した。

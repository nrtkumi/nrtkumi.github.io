# MEZAME -1/8192-

2つの扉から1つを選び続ける、確率1/8192の運の試練。あなたは夢の最深層「第13層」で眠っており、**当たりの扉を13回連続で選べば目覚める**。外せば最深層へ墜落して最初からやり直し。連勝が伸びるほど世界が明るくなり、演出と音がエスカレートしていく。**three.jsによる3D空間**で、頭上に積み重なる13層の光輪を正解のたびに突き抜けて浮上し、墜落時は世界が逆流する。

- **プレイURL**: https://nrtkumi.github.io/games/mezame/
- **対応環境**: スマホ(縦画面・タッチ操作)ファースト。PCのマウス・キーボード操作にも対応。単一HTML(three.jsのみCDNから読み込み)。

## 遊び方

1. 「眠りにつく」をタップ。第13層からスタート
2. 並んだ2つの扉のどちらかをタップ(PC: ←→/A・D)。当たりは毎回ちょうど1/2
3. **当たり**なら1層浮上。**ハズレ**なら第13層へ墜落し、連勝はゼロに
4. 13連勝(確率 1/8192)で「目覚め」── クリア

### 十三層の夢

連勝するごとに層を浮上し、層ごとに名前と色彩が変わる(忘却の底 → 沈黙の海 → 骨の庭 → 鏡の沼 → 月の回廊 → 囁きの森 → 逆さの街 → 硝子の塔 → 黄昏の駅 → 暁の海岸 → 光の梯子 → 白い部屋 → まぶたの裏 → 目覚め)。

- **夢日記**: 各層に初めて辿り着くと、その層の記憶(短い散文)が解禁される収集要素。全14エントリ
- **演出のエスカレーション**: 連勝が伸びるほど、選択後のタメ(サスペンス)が長くなり、心拍音が速まり、扉が震え、正解時のフラッシュ・パーティクル・上昇音の音程が派手になっていく
- **確率の可視化**: 「ここまで 1/256 ／ 目覚めまで 1/32」のように現在地の確率を常時表示。通算何枚目の扉かのカウンターも永続化
- 高層からの墜落には専用の煽りメッセージあり(第1層で落ちると「目覚めの直前で、夢に呑まれた。」)

## 作成の背景(市場調査の要点)

ユーザーからのリクエスト「2分の1を選択し続けるゲーム」+参考作品の仕様調査をもとに企画した(2026年6月)。

- **参考作品『1/8192』(Steam)**: 青と緑の2つのゲートを選び、当たりを13回連続で引くと涅槃到達(1/8192)。外すと最初のステージに戻る。仏教曼荼羅の世界観の一人称ウォーキングシム。¥350で「非常に好評」(142件・84%肯定)。クリア率7.4%という報道もあり、「純粋な1/2を積み重ねるだけ」という極端なシンプルさがフックになっている([Steamストア](https://store.steampowered.com/app/3014150/18192/?l=japanese)、[高校生新聞の紹介記事](https://www.koukouseishinbun.jp/articles/-/11898))
- **casino-core / push-your-luckの隆盛**: Balatro以降、ギャンブルの心理リズムを実通貨なしで再構成するインディーが一大潮流([Game Wisdom](https://game-wisdom.com/general/indie-games-turned-gambling-new-trend)、[GameTyrant](https://gametyrant.com/news/random-chance-systems-in-game-design-2026))→ 本作は「確率の階段を登る快感と墜落の喪失感」に全振り
- **差別化**: 同サイトの『うんだめし』(おみくじpush-your-luck)とは「降りる判断があるチキンレース」vs「降りられない連続二択」で駆け引きの質を変えた。世界観も元ネタの仏教モチーフを避け、「13層の夢から目覚める」というオリジナル設定にして夢日記の収集で再挑戦の動機を作った。**確率はいっさい補正せず常に1/2**(そこが元ネタの核のため)

## 技術構成

- 単一HTML + **three.js(CDN/importmap、r170)による3D空間**。Pointer Events、clickイベント不使用
- **ロジック層とビュー層を分離**: ゲーム状態・進行・音・永続化・DOM HUDは素の`<script>`(従来どおりNode vmでテスト可能)、three.jsの3D描画はESモジュールが `window.__MEZ_BRIDGE__`(state/pal/fxキュー)を毎フレーム読むだけの純粋なビュー。**CDNが読めない環境でも、CSSグラデ背景+DOM HUDでゲーム自体は成立する**フォールバック構造
- 3Dシーン: グラデーション天球(ShaderMaterial)+霧、頭上に積み重なる13層の光輪(通過済みは層のアクセント色で点灯)、発光チューブフレームのアーチ扉(紋章はCanvasTexture)、加算ブレンドの漂う夢の粒・勝利バースト・墜落スピードライン。上昇/墜落は光輪グループの縦オフセットをイージングで動かし「世界が流れる」を表現
- 扉の選択はレイキャストではなく**画面の左半分/右半分**で判定(扉は左右対称なので3Dに依存せず、フォールバック時も同じ操作が成立)
- HUD(層番号・オーブ・確率・バナー・FALL表示)はDOMで3Dの上に重ね、CSSアニメーションで演出。サスペンスのビネット暗転と正解フラッシュもDOMオーバーレイ
- フォントは既存ページと統一(Bungee + Zen Kaku Gothic New + Space Mono)
- 効果音はWeb Audioでリアルタイム合成: 連勝数に応じて基音が半音ずつ上がる正解アルペジオ、サスペンス中の心拍(間隔が徐々に詰まる)、墜落のベースドロップ、目覚めのファンファーレ。層が上がるほど音程が上がる通奏低音ドローンつき
- **iOS消音スイッチ対策**: 初回タップ時に無音WAV(JSでArrayBuffer生成→Blob URL)を`<audio>`要素でループ再生してオーディオセッションを再生カテゴリへ切り替え、`AudioContext.resume()`のPromise完了を待ってアンロック判定、`onstatechange`で中断から自動復帰(games/hakai/と同方式)
- メインループ(rAF)はタイトル画面の時点から回し、ループが触る状態は全て宣言時に初期化
- ダブルタップズーム無効化の3点セット(`touch-action: manipulation` + dblclick/gesturestart抑止 + 連続touchend抑止)
- PCワイド画面ではルートを `min(100vw, 520px, 64vh)` の縦長カラムに制限して中央寄せ。キーボード操作(←→/A・D 選択、Space/Enter 開始・スキップ)対応
- 通算選択数・墜落数・ベスト連勝・目覚め回数・夢日記の解禁ビット・ミュート設定はlocalStorage(try/catchでフォールバック)
- 層のパレット(背景グラデ+アクセント色)は遷移中に補間し、上昇/墜落でシームレスに色が変わる
- テスト用に読み取り+操作フック `window.__MEZ__` を公開(state/doorRect/start/pick/rig/skipSuspense/skipAnim)。`rig()`で次の結果を強制でき、決定論的にテストできる
- **キービジュアル2枚はCodex CLI($imagegenスキル / gpt-image-2)で生成**: タイトル用(藍色の夢の最深層に浮かぶ2つの扉と13層の光輪)と目覚め用(カーテン越しの金色の朝日)。1024px PNG生成→`sips`で512px JPEGに変換、読み込み失敗時は`onerror`で非表示にフォールバック
- OGP画像(1200x630)はキービジュアルをベースに合成

## 実施したテスト

1. **構文チェック**: ロジック層scriptとビュー層モジュール(.mjs)をそれぞれ抽出して `node --check` → OK
2. **ヘッドレスロジックテスト**: DOM/localStorageをスタブしてNode(vm)でロジック層を駆動。タイトル時点のループ生存→開始→13連勝で目覚め(streak増加・夢日記全解禁・記録保存)→ブリッジ(state/pal/fxキュー上限)→再スタート→墜落(fallFrom・正解側・墜落数)→suspense中の入力無効→実乱数30回プレイ、まで全55項目PASS
3. **実ブラウザE2E**: puppeteer-core + ローカルChrome、**ローカルHTTPサーバ経由**、モバイルビューポート(390×844、`hasTouch: true`)+タッチイベントで実プレイ。pageerror/consoleエラーゼロ、three.jsのWebGL描画発生(`renderer.info.render.calls`)、タッチで開始→扉選択→サスペンス→勝利→墜落→13連勝で目覚め→夢日記14エントリ→リロード後の記録復元、まで全22項目PASS。デスクトップ(1440×900)でも縦長カラム制限とキーボード操作を確認。プレイ中・中層・サスペンス・上昇・墜落の各スクリーンショットを目視確認

### 開発で得た教訓

- 「最高到達 第N層」のような逆順の表示(層番号 = 13 - 連勝)は、端の値(連勝13 → 第0層)で破綻しやすい。クリア状態は専用表記に分岐する
- サスペンス演出の長さを連勝数でスケールさせる(0.85s + 0.11s/連勝、上限2.1s)と、序盤はテンポよく、終盤は勝手に「タメ」が効いて脳汁が出る。固定長より体感がかなり良い
- **縦画面の3Dは水平視野が極端に狭い**。縦844×横390ではfov 55°でも水平視野は約27°しかなく、PC前提の距離感で置いたオブジェクトは確実に見切れる。横に並ぶ要素の幅から必要な水平視野を逆算してカメラ距離を決める(本作はfov 62°・距離11.5でようやく2枚の扉が収まった)
- 3Dビューを「状態を毎フレーム読むだけの純粋関数」+少量のfxイベントキューに分離すると、ロジックのヘッドレステストがそのまま生き、3D側のフェーズ同期バグ(イベント取りこぼし)も起きない。上昇/墜落のオフセットはイベントではなくphase+phaseTから決定的に導出するのがコツ

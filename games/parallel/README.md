# パラレル株式会社 — 並行世界同期課 (PARALLEL Inc.)

2つの並行世界を、**1本の指で同時に操作する**ローグライト・アクション。
あなたは「パラレル株式会社 並行世界同期課」の社員。上下に並ぶ **世界A** と **世界B** を同時に動かし、落ちてくる **締切ウォール** の隙間を両世界くぐり抜け、**契約書** を集めて決算ごとに昇進(強化)していく。

- **プレイURL**: https://nrtkumi.github.io/games/parallel/
- **対応環境**: スマホ / PC ブラウザ。依存なしの単一HTML(外部CDNはWebフォントのみ)。タッチ・マウス・キーボード対応。

---

## 遊び方

1. **出社する** を押すとゲーム開始。画面は上下2分割 — 上が **世界A**(琥珀)、下が **世界B**(シアン)。
2. **画面を左右にドラッグ**(PCは ←→ / A・D キー)すると、**両世界のキャラが同じ列へ同時に移動**する。これがこのゲームの肝。
3. 赤い斜線の **締切ウォール** は、空いている1〜2列の **隙間だけが安全**。両世界のキャラを、それぞれの隙間に通せ。
4. **契約書📄** を拾うと売上(スコア)アップ。**金の契約** は高得点。
5. **両世界でほぼ同時に契約を拾う** と **並行ボーナス**(コンボ)が発生し、倍率が跳ね上がる。
6. 締切ウォールにぶつかると体力が減る。体力0で **倒産(ゲームオーバー)**。
7. 一定時間ごとに **決算** が来て、**昇進** として3つの中から強化を1つ選ぶ。強化はそのランの間ずっと有効。

### コアの面白さ

2世界の操作は **連動している**(同じ列に動く)が、ウォールの出現タイミングは世界ごとにズレている。
だから「世界Aの隙間に合わせて動く → ウォールが過ぎる前に世界Bの隙間へ動く」という、**1入力で2つの制約を同時に満たすパズル的なアクション**になる。ウォールは必ず通れるよう時間差で出るので理不尽さはなく、純粋に「同時に2つを見る」脳の忙しさが中毒性になる。

## 作成の背景(市場調査の要点)

`/deep-research` ハーネス(複数ソースをファンアウト検索し、各主張を3票の敵対的検証にかけて統合)で調査し、以下2つの実証済みの潮流を組み合わせた:

- **「1入力で複数を同時操作/同時注視する」メカニクス** — 最も直接的な先行例は `Parallyzed`(2017):双子姉妹を**同時に操作**し、いつでも身体を**スワップ**できる。操作は「ジャンプ」と「スワップ」の**2タッチだけ**で、各キャラが固有属性(色・サイズ)を持つため、次元ごとの障害に合わせて絶え間なくスワップ判断を迫られるのが中毒性の源泉([Steam](https://store.steampowered.com/app/545670/Parallyzed/) / [AppAdvice](https://appadvice.com/review/parallyzed))。`You Must Build A Boat`(2015)は厳密な同時操作ではないが「画面の2領域を**同時に注視**させ、1つの入力が複数システムを駆動する」設計の典型([Steam](https://store.steampowered.com/app/290890/You_Must_Build_A_Boat/))。ワンボタン/タップ操作は「画面のどこでもタップ=巨大ボタン、反応の悪い仮想パッドを回避、説明不要」でタッチ最適という指摘もある(ABA Games / Kenta Cho、[一ボタンゲーム論](https://abagames.github.io/joys-of-small-game-development-en/restrictions/one_button.html))。中毒性の理論的裏付けは**フロー理論**(challenge≈skill のスイートスポット=易しすぎる退屈と難しすぎる不安の間)。
- **ハイブリッドカジュアル × ローグライト強化** — 2024-2025の支配的パターンは「ハイパーカジュアルのシンプルなコア+カジュアルのメタ(ブースター/進行レイヤー/シナジー)」の重ね合わせで、ハイブリッドカジュアルは2024年に最速成長。ジャンル構成も Arcade 偏重(2023 Q1=77%)から Puzzle 寄り(2025 Q1=48%)へシフト([AppMagic Q1 2025](https://appmagic.rocks/blog/hybridcasual-q1-2025/))。`You Must Build A Boat` 自身も「手続き生成+ラン間で永続化するメタ目標(クルー勧誘・強化・ショップ拡張)」というローグライト層を持つ。

研究結果が示すのは「**パラレル操作**」と「**ローグライト層**」は独立して組み合わせ可能ということ。本作は `Parallyzed` 型の同時操作を「縦画面・複数の並行世界の同時運営」に翻訳し、ラン制の決算強化(3択シナジー)を被せた。これを「**パラレル株式会社で複数の並行世界を同時に回す社員**」という会社員コメディの世界観に落とし込み、締切=ウォール、契約=スコア、決算=強化選択、昇進=ローグライト進行、というメタファーで統一している。

> 出典は上記リンクの通り(Parallyzed=同時操作+2タッチ・Steam一次情報、You Must Build A Boat=2領域同時注視+ローグライト・メタ、AppMagic=ハイブリッドカジュアル/ジャンルシフトの統計、ABA Games=ワンボタン設計論)。

## デザインコンセプト

**「スプレッドシート / オフィス端末 × 並行世界のグリッチ」**。

- 配色: インク紺の背景 + 方眼グリッド、世界A=琥珀(マニラ封筒)、世界B=シアン、ハザード=ハンコ赤。AI生成にありがちな紫グラデ・グラスモーフィズム・ネオン一辺倒は避けた。
- タイポgrafi: 見出し・数値は `DotGothic16`(ドット文字のレトロ業務端末風)、本文は `Noto Sans JP`。
- モチーフ: 締切=赤い斜線のハザードバー、契約=ハンコ付きの書類、社員=ネクタイ姿のドット会社員、決算=昇進カード。

## 技術構成

- **単一HTMLファイル**。描画は canvas 2D を手書き(ライブラリ非依存)。Webフォントのみ CDN から読み込み。
- 効果音は **Web Audio API でリアルタイム合成**(外部音源なし)。並行ボーナス時はコンボ数に応じてピッチが上がる。
- **iOS消音スイッチ対策**: 初回タップで無音WAVを `<audio>` ループ再生してオーディオセッションを再生カテゴリへ切替 + `AudioContext.resume()` の Promise 完了待ち + `onstatechange` での `interrupted` 自動復帰。
- **スマホ対応**: `viewport-fit=cover` + セーフエリア、`touch-action` 制御、ダブルタップズーム防止(`dblclick`/`gesturestart`/350ms以内の連続`touchend` を preventDefault、入力は Pointer Events に統一)。PC幅では `max-width: min(480px, 62vh)` の縦長カラムに制限して中央寄せ。
- **メインループ(rAF)はタイトル時点から起動**し、ループが触る状態は全て宣言時に初期化(初回フレーム例外でループを殺さない設計)。`update`/`draw` は try/catch で保護。dpr変換は毎フレーム `ctx.setTransform` で再設定(iOS Safari のコンテキスト喪失対策)。
- スコア・最高売上・勤続日数は `localStorage` に保存(try/catch フォールバック)。
- 乱数は決定論的LCG(`Math.random` 非依存)でヘッドレステストを安定化。シードはラン開始時の時刻で攪拌。
- OGP画像: 実プレイ画面のポートレート・スクリーンショットを Puppeteer で撮影し、1200×630 のスマホ枠カードに合成(`assets/ogp.jpg`)。

## 実施したテスト

1. **構文チェック**: インラインscriptを抽出して `node --check`。
2. **ヘッドレスロジックテスト**: DOM/canvas/AudioContext/localStorage をスタブし、Nodeでゲームループを駆動。開始→操作→ウォール判定→契約取得→決算で強化選択→3四半期到達→世界3つへ増設→ゲームオーバー→localStorage保存、までを通過確認。
3. **実ブラウザ E2E(Puppeteer + ローカルChrome、モバイルビューポート `hasTouch`、HTTPサーバ経由)**: `pageerror`/console エラーが **0件**、タイトル/プレイ/決算の各画面で canvas に描画ピクセルがあること、タッチ操作でスコア・契約・ウォールが動くこと、決算画面に強化カードが3枚生成されることを確認。スクリーンショットで見た目も目視確認。

### 開発で得た教訓

- 連動操作ゲームは「必ず通れる」担保が命。世界ごとにウォール出現を時間差にすることで、1入力でも両世界を必ず捌けるフェアさを確保した。
- HUDのオーバーレイ(休憩ボタン)とcanvas描画(体力表示)の座標衝突は実機スクショで初めて気づける。E2Eのスクリーンショット目視は省略しない。

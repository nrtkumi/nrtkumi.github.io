# パラレル オールナイト — たまり場の熱量を朝まで守れ

Z世代向け音声SNS **「パラレル」(パラレル株式会社)** を題材にした、たまり場ホスト・ローグライト。
あなたは深夜のオンラインたまり場の**ホスト**。集まった親友たちの**「今やりたい気分」**(🎮ゲーム / 🎬動画 / 🎤カラオケ / 🎵音楽 / 💬だべり)を読み、ルームの**アクティビティを切り替えて**、**コミュニケーション熱量**を朝まで保ち続ける。

- **プレイURL**: https://nrtkumi.github.io/games/parallel/
- **対応環境**: スマホ / PC ブラウザ。依存なしの単一HTML(外部CDNはWebフォントのみ)。タップ・マウス・キーボード(1〜5 / Space)対応。

> 題材は実在のサービス。パラレルは「友達とゲーム・動画・音楽などを一緒に楽しむオンラインのたまり場アプリ」で、グループ通話しながらUNO等のミニゲーム・YouTube同時視聴・**カラオケ機能**・**ミュート(親フラ対策)**などを備えるZ世代の音声SNS。利用者の7割がZ世代、1日平均通話は約3時間。企業ミッションは「世界中のコミュニケーション熱量を高め、好きな人たちとの時間を最大化する」。([Nikkei xTrend](https://xtrend.nikkei.com/atcl/contents/18/00533/00002/) / [MarkeZine](https://markezine.jp/article/detail/39047) / [Real Sound](https://realsound.jp/tech/2021/04/post-734076.html))

## 遊び方

1. **通話をはじめる** を押すと、たまり場に親友が集まる。各友達の頭上の**吹き出し**が「今やりたい気分」。
2. 画面下の**アクティビティバー**(🎮🎬🎤🎵💬)をタップして、ルームのアクティビティを選ぶ(PCは数字キー 1〜5)。
3. 気分が一致した友達は**盛り上がり**、まわりに ❤️🎶 が舞い、**コミュニケーション熱量**(上部ゲージ)が上がる。
4. 気分が合わない友達は**退屈**してvibeリングが減り、ゼロになると**ログオフ**して熱量が下がる。全員の熱量(上部ゲージ)が尽きると**解散(ゲームオーバー)**。
5. **親フラ📵**が来た友達はタイマー内に**タップしてミュート**!放置するとログオフ&熱量大ダウン。**眠気😴**もタップで起こす。
6. 熱量が満タンになると**パーティータイム(フィーバー)**!全員が一時的に大盛り上がり、スコア2倍。
7. ゲーム内で**1時間進むごと**に「アップデート」が来て、3つの強化から1つ選ぶ。**朝6:00**まで持たせれば完徹達成、その後も熱量が尽きるまでスコアアタックは続く。

### コアの面白さ ─ 「場を読んで、回す」

親友はそれぞれ好きなアクティビティが違い、気分は数秒ごとに移り変わる。**1つのアクティビティで全員は満たせない**から、「今いちばん多くの友達が求めているもの」を読み、刻々と切り替え続ける。人数が増えると一人では満たしきれず、**画面共有(同時に2アクティビティ)**などのシナジーで切り抜ける。これは「複数の親友の今の気分に合わせて1つの行動を選ぶ」というたまり場ホストならではの判断遊びだ。

## 作成の背景(市場調査の要点)

題材(パラレル社/アプリ)を一次情報で確認したうえで、`/deep-research` ハーネス(複数ソースをファンアウト検索→各主張を3票の敵対的検証→統合)でゲームメカニクスを調査し、以下の実証済みパターンを組み合わせた:

- **「ムードメーターを保つ失敗回避ループ」** — ゲスト個々に気分メーターを持たせ、減衰→離脱、離脱が一定数で解散、という構造はホスト/接客ゲームの王道(`Just Get The Party Started`、`Diner Dash`、`Happy Crowd`)。本作の「熱量ゲージ × 友達のvibe × ログオフ」はこれを踏襲。
- **「読み合わせ・相性管理」が面白さの核** — ゲスト種ごとに好みが違い、複数の状態を読んで正解の1行動を選ぶ判断が中毒性を生む(`Happy Crowd`「誰が誰を許容できるか・どの行動が誰に効くか」、`Diner Dash` の客種ごとの我慢強さ)。パラレルの「複数の親友の気分に合わせて場を回す」と完全一致。
- **ローグライトの中毒構造** — ラン間メタ強化+難度再付与(`Hades`)、3択シナジーで「自己完結する得点マシン」を組む金欠ジレンマ(`Balatro`)、数字が雪崩を打つ Rube-Goldberg 的フィーバー演出。本作の「1時間ごとの3択アップデート」「画面共有×カラオケ等のシナジー」「パーティータイム」に反映。
- **スマホ縦・片手タップの実証** — `UnderKnight`(完全縦・片手・少数ジェスチャー)、`Coffee Caravan`(移動カフェ運営×ローグライトのコジー作)など、運営×ローグライト×縦タップが成立する事例を確認。

## 技術構成

- **単一HTMLファイル**。描画は canvas 2D を手書き(ライブラリ非依存)。Webフォントのみ CDN から読み込み。
- 効果音は **Web Audio API でリアルタイム合成**(外部音源なし)。パーティータイムは上昇アルペジオで演出。
- **iOS消音スイッチ対策**: 初回タップで無音WAVを `<audio>` ループ再生 + `AudioContext.resume()` の Promise 完了待ち + `onstatechange` の `interrupted` 自動復帰。
- **スマホ対応**: `viewport-fit=cover` + セーフエリア、`touch-action` 制御、ダブルタップズーム防止(`dblclick`/`gesturestart`/350ms以内の連続`touchend` を preventDefault、入力は Pointer Events)。PC幅では `max-width: min(480px, 64vh)` の縦長カラムに制限して中央寄せ。
- **メインループ(rAF)はタイトル時点から起動**、ループが触る状態は宣言時に全初期化(初回フレーム例外でループを殺さない)。`update`/`draw` は try/catch で保護。dpr変換は毎フレーム `ctx.setTransform`(iOS Safari のコンテキスト喪失対策)。
- スコア(熱量pt)・最高熱量・通算夜数は `localStorage` に保存(try/catch フォールバック)。
- 乱数は決定論的LCG(`Math.random` 非依存)でヘッドレステストを安定化。
- OGP画像: 実プレイ画面(5人接続・パーティータイム)のポートレート・スクショを Puppeteer で撮影し、1200×630 のスマホ枠カードに合成(`assets/ogp.jpg`)。

## 実施したテスト

1. **構文チェック**: インラインscriptを抽出して `node --check`。
2. **ヘッドレスロジックテスト**: DOM/canvas/AudioContext/localStorage をスタブし、Nodeでゲームループを駆動。開始→アクティビティ切替→満足/退屈→ログオフ→パーティータイム→1時間ごとの強化選択→**死亡時のlocalStorage保存**、画面共有の2アクティビティ上限、招待での人数増、を確認。
3. **バランス調整(ホストボットのシミュレーション)**: 完全情報の「読み合わせ」ボットを反応速度・正確さを変えて実行。**完璧/上手い/カジュアル(0.9s・80%正答)なホストは朝6:00に到達**してスコアアタックへ、**雑な(1.4s・60%)ホストは04:00で解散**、と腕前が結果に反映されるフェアな曲線を確認。これを受けて開始vibe・退屈の減衰を調整して序盤の理不尽さを除去した。
4. **実ブラウザ E2E(Puppeteer + ローカルChrome、モバイルビューポート `hasTouch`、HTTPサーバ経由)**: `pageerror`/console エラーが **0件**、タイトル/プレイ/強化の各画面で canvas に描画ピクセル、タップでアクティビティ切替・熱量・スコアが動くこと、強化カードが3枚生成されることを確認。スクリーンショットで見た目も目視確認。

### 開発で得た教訓

- 「全員を同時に満たせない」緊張が核なので、**退屈の減衰が速すぎると序盤が理不尽**になる。開始vibeを上げ減衰を緩めて、初手の反応猶予を確保した(ボットシムで数値検証)。
- 時刻が 21:00→翌6:00 と日付をまたぐため、**「夜の深さ」は `hour>=21? hour-21 : hour+3` で 0..9 に正規化**しないと、22:00で「もうすぐ朝」と出るようなバグになる(実際に踏んで修正)。
